# 두 번째 중간 보스 군체 수직 슬라이스 설계

## 목표

첫 중간 보스 보상 뒤 일반 전투만 무한히 이어지는 현재 런에 두 번째 고저점을 만든다. 첫 보스와 다른 판단을 요구하는 군체형 보스, 더 강한 두 번째 보스 유물, 세 번째 구간 재개까지 구현해 한 판의 성장 곡선이 실제로 이어지는지 검증한다.

이번 작업은 다음을 포함한다.

- 첫 보스 보상 선택 뒤 시작되는 두 번째 일반 전투 구간
- 위협 단계 3부터 등장하는 분열형 특수 적
- 성과와 시간을 함께 사용하는 두 번째 보스 진입
- 중앙 코어와 네 모듈로 구성된 군체형 보스
- 두 번째 보스 전용 상위 유물 풀
- 두 번째 보상 선택 뒤 세 번째 일반 전투 구간 재개

최종 보스, 스테이지 클리어, 메타 해금, 정식 그래픽과 사운드는 포함하지 않는다.

## 런 진행

진행 순서는 다음과 같다.

```text
첫 보스 처치
-> 첫 보스 유물 선택
-> 두 번째 구간 시작(section 1, 구간 시간 0)
-> 두 번째 보스 경고
-> 군체형 보스
-> 두 번째 상위 유물 선택
-> 세 번째 구간 시작(section 2, 구간 시간 0)
```

보스 처치 시점이 아니라 보상 선택을 완료해 전투가 재개되는 순간 다음 구간 시계를 0으로 시작한다. 보상 선택 중에는 구간 시간, 적 생성, 보스 공격 시계가 모두 정지한다.

다음 값은 구간을 넘어 유지한다.

- 플레이어 현재 체력
- 일반 능력 등급
- 영구 구슬 상태와 한도
- 이미 획득한 보스 유물

다음 값은 새 구간이 시작될 때 초기화한다.

- 구간 경과 시간
- 해당 구간 보스 처치 점수
- 일반 적 보충 생성 시계
- 보스 경고 시간과 보스 객체

두 번째 보상 뒤에는 세 번째 일반 구간을 즉시 시작하지만 최종 보스 조건은 이번 작업에서 활성화하지 않는다.

## 보스 이후 위협 단계

보스 보상으로 플레이어가 강해졌는데 일반 적 난도가 이전 단계로 낮아지는 흐름을 허용하지 않는다.

- 첫 구간 후반: 기존 위협 단계 2
- 첫 보상 뒤 두 번째 구간: 위협 단계 2에서 즉시 시작
- 두 번째 구간 `60000ms` 이후: 신규 위협 단계 3
- 두 번째 보상 뒤 세 번째 구간: 위협 단계 3에서 즉시 시작

신규 위협 단계 3의 초기값은 다음과 같다.

- 대형 적 수: `21~25`
- 활성 적 상한: `84`
- 보충 생성 간격: `5500ms`
- 대형당 장갑 적: `3`
- 대형당 사수 적: `3`
- 대형당 분열형 적: `2`

일반 적 하강 속도와 종류별 HP는 현재 값을 유지한다. 위협 단계 3은 단순 체력 배율이 아니라 적 수, 특수 적 비율, 생성 압박으로 난도를 올린다. 대형 생성 규칙은 기존 절차형 생성기를 그대로 재사용한다.

## 분열형 특수 적

위협 단계 3부터 분열형 부모 적을 대형마다 2마리 배치한다.

### 부모

- HP: `3`
- 초기 크기: `38x30px`
- 하강 속도: 현재 일반 적 하강 속도와 동일
- 처치 시 보스 진행 점수: `2`
- 처치 경험치: `1`
- 최하단 돌파 피해: `3`
- 활성 적 자리 비용: `2`

### 파편

- 부모가 파괴된 위치의 좌우 `12px`에 2개 생성
- HP: 각각 `1`
- 초기 크기: `22x18px`
- 하강 속도: 부모와 동일
- 재분열 없음
- 처치 시 보스 진행 점수: `0`
- 처치 경험치: 각각 `1`
- 최하단 돌파 피해: 각각 `1`
- 활성 적 자리 비용: 각각 `1`

부모와 파편은 다른 적과 겹칠 수 있다. 빈자리 탐색, 주변 적 밀어내기, 생성 지연은 사용하지 않는다. 파편 중심 좌표는 파편 반너비만큼 화면 안으로 clamp해 월드 경계 밖이나 경계 벽 내부에 생성되지 않게 한다. 벽 가까이에서 두 파편끼리 일부 겹치는 것은 허용한다.

부모가 살아 있는 동안 활성 적 자리 2칸을 예약한다. 부모 하나가 파편 둘로 바뀌어도 활성 자리 합계가 변하지 않으므로 위협 단계의 활성 상한을 넘지 않는다. 일반 적 보충 가능 여부는 단순 스프라이트 수가 아니라 이 활성 자리 비용 합계를 사용한다.

부모 처치와 파편 생성은 하나의 피해 정산에서 정확히 한 번만 발생한다. 파편은 부모를 죽인 동일 피해 이벤트의 대상이 되지 않는다. 생성 이후의 다른 구슬, 폭발, 지속 피해에는 정상적으로 맞는다. 파편 처치 점수는 0으로 두어 분열 자체가 두 번째 보스 진입 점수를 부풀리지 않게 하지만, 모두 정리하면 총 경험치 3을 얻어 추가 위협에 대한 보상은 유지한다.

프로토타입 텍스처는 부모 중앙의 갈라진 선과 좌우 파편의 서로 맞물리는 실루엣으로 기본·장갑 적과 구분한다. 정식 테마와 색은 그래픽 작업에서 바꿀 수 있지만, 분열 전후 관계는 현재 단계에서도 읽혀야 한다.

## 두 번째 보스 진입

두 번째 구간은 생성된 적 수가 아니라 처치한 적의 위협 점수를 사용한다. 기존 적은 첫 구간과 동일하게 기본 적 `1`, 장갑 적과 사수 적 `2`를 사용한다. 분열형 부모는 `2`, 파편은 `0`을 사용한다.

- 최소 등장 시간: `150000ms`
- 목표 처치 점수: `110`
- 강제 등장 시간: `210000ms`
- 경고 시간: `2000ms`

구간 시간 `150초` 이후 목표 점수를 달성하면 즉시 보스 경고를 시작한다. 점수가 부족해도 `210초`에 도달하면 강제로 경고를 시작한다. 경고와 보스 전투 중에는 일반 보충 대형을 생성하지 않는다.

이 조건은 첫 구간의 숫자를 복사한 분기문으로 만들지 않는다. 중앙 설정의 구간별 보스 일정이 보스 종류, 최소 시간, 점수, 강제 시간을 함께 소유한다.

## 군체 보스 구성

두 번째 보스는 중앙 코어와 네 모듈로 구성한다.

- 중앙 코어: `72 HP`
- 사격 모듈 2개: 각각 `12 HP`
- 반사벽 모듈 2개: 각각 `14 HP`
- 목표 전투 시간: `45~75초`

코어 HP는 플레이어 레벨이나 공격력에 따라 자동 보정하지 않는다. 강한 빌드는 보스를 더 빨리 처치할 수 있어야 한다. 난도 상승은 체력 팽창보다 공격 조합과 공 경로 변화에서 만든다.

모듈은 각자 별도 HP를 가지며 파괴되면 해당 런에서 영구 제거된다. 코어만 집중 공격해 보스를 끝낼 수도 있다. 네 모듈을 모두 파괴하면 보호막을 다시 만들 수 없고 코어는 영구 노출된다.

## 결합과 전개 주기

군체 보스는 다음 주기를 반복한다.

```text
결합 보호막 4000ms
-> 전개 예고 1500ms
-> 모듈 전개와 코어 노출 7000ms
-> 생존 모듈 복귀
-> 결합 보호막
```

- 결합 상태에서는 코어가 피해를 받지 않고 영구·임시 구슬을 정상 반사한다.
- 전개 예고는 시각 효과와 소리 훅을 모두 제공할 수 있는 별도 상태로 둔다.
- 전개 중에는 코어가 완전히 노출되고 받은 피해는 다음 주기에도 유지된다.
- 모듈은 결합과 전개 상태 모두 피해를 받을 수 있다.
- 파괴된 모듈은 복귀하거나 재생되지 않는다.
- 네 모듈이 모두 파괴되면 현재 상태와 관계없이 코어를 영구 노출 상태로 전환한다.
- 레벨업, 보스 보상, 창 숨김 일시정지 중에는 주기 시간이 흐르지 않는다.

`7초` 노출은 이미 발사된 공이 돌아오는 게임 특성을 고려한 최소 기회다. 공의 현재 위치에 따라 노출 종료를 연장하거나 공을 강제로 회수하지 않는다. 대신 긴 예고와 반복 노출로 타이밍 실패가 한 번의 기회 상실에 그치게 한다.

## 사격 모듈

사격 모듈은 전개 시 코어의 상단 좌우로 갈라진다.

- 각 모듈 발사 간격: `1400ms`
- 좌우 첫 발사 오프셋: `700ms`
- 조준 예고: `300ms`
- 피해: `1`
- 초기 탄속: `170px/s`

각 모듈은 예고가 끝날 때의 플레이어 위치가 아니라 예고 생성 시점의 플레이어 위치를 고정해 조준한다. 두 모듈은 같은 프레임에 발사하지 않고 엇갈려 공격한다. 하나가 파괴되면 해당 방향과 해당 모듈의 발사 시계가 사라져 실제 공격 빈도가 낮아진다.

모든 사격은 기존 일반 적과 보스가 사용하는 전역 적대 탄환 상한을 공유한다.

## 반사벽 모듈

반사벽 모듈은 전개 시 상단과 화면 중앙 사이에 세로 벽을 형성하고 천천히 좌우로 이동한다.

- 초기 벽 크기: `18x96px`
- 영구 구슬과 임시 구슬은 정상 반사된다.
- 벽에 부딪힌 구슬은 해당 모듈에도 직접 피해를 준다.
- 플레이어와 적대 탄환은 벽을 통과한다.
- 반사 재충돌 제한: 같은 구슬과 같은 모듈 조합에 `80ms`
- 코어, 다른 모듈, 화면 경계와 겹치는 이동 위치는 허용하지 않는다.

벽은 공을 없애거나 강제 회수하지 않는다. 이동 속도와 범위는 중앙 설정에 둔다. 두 벽 사이에서 같은 구슬이 장시간 갇히지 않도록 최소 통로 폭을 보장하고 재충돌 제한을 적용한다. 좋은 각도에서는 벽 반사를 이용해 열린 코어를 여러 번 타격할 수 있어야 한다.

## 중앙 코어 공격

코어는 모듈 전개 한 번마다 느린 5발 확산탄을 한 차례 발사한다.

- 발사 각도: 아래쪽 기준 `-36`, `-18`, `0`, `18`, `36`도
- 피해: `1`
- 초기 탄속: `140px/s`
- 넓은 탄 사이 간격을 유지해 이동 가능한 경로를 남긴다.

모듈을 모두 파괴한 뒤에도 코어 확산탄은 주기적으로 유지한다. 영구 노출 상태에서는 기존 전개 주기 길이인 `7000ms`마다 한 번 발사한다. 따라서 모듈 전멸이 전투 압박을 완전히 제거하지는 않는다.

두 번째 보스 전투 중 일반 적 증원은 없다. 경고 시작 전에 이미 있던 일반 적은 첫 보스와 동일하게 유지한다.

## 두 번째 보스 유물

두 번째 보스 처치 시 전투를 완전히 정지하고 상위 유물 3개 중 1개를 선택한다. 첫 보스 유물과 별도 풀을 사용하고 같은 상위 유물을 중복 획득할 수 없다.

상위 풀은 범용 유물 3개와 능력 진화 유물 4개로 구성한다.

### 범용 유물

- `보조 궤도`: 영구 구슬 한도 `+1`. 전체 상한 `6`을 넘지 않는다.
- `회수 일제사`: 근접 직접 회수한 영구 구슬을 재발사할 때 좌우로 임시 구슬 2개를 함께 발사한다. 이 임시 구슬은 현재 임시 구슬 수명과 전역 상한을 사용한다.
- `공성 공명`: 영구 구슬 직접 적중 10회를 누적하면 다음 영구 구슬 직접 적중에서 반경 `80px`, 피해 `2`의 충격파를 한 번 발생시킨다. 충격파는 적중 횟수를 추가하지 않고 다른 공성 공명을 재귀 발생시키지 않는다.

### 능력 진화 유물

- `초고압 탄심 / 화력`: 화력 등급 1 이상에서만 등장한다. 충전 상태 영구 구슬의 직접 피해에 `+0.75`를 더한다.
- `관성 관통 / 속도`: 속도 등급 1 이상에서만 등장한다. 충전 영구 구슬이 직접 피해로 적을 처치하면 반사하지 않고 기존 방향과 충전 속도를 유지한다. 충전 소모는 정상 적용한다.
- `잔향 폭발 / 폭발`: 폭발 등급 1 이상에서만 등장한다. 영구 구슬이 만든 폭발 위치에 `350ms` 뒤 기존 반경의 `80%`, 기존 피해의 `50%`인 잔향 폭발을 한 번 만든다. 잔향은 폭발·잔향·분열을 다시 발생시키지 않는다. 임시 구슬 폭발에는 적용하지 않는다.
- `연쇄 분열 / 분열`: 분열 등급 1 이상에서만 등장한다. 자식이 아닌 임시 구슬이 첫 적 직접 적중 시 진행 방향 기준 `-25`, `25`도로 자식 임시 구슬 2개를 한 번 생성한다. 자식은 재분열하지 않으며 현재 전역 임시 구슬 상한을 넘지 않는다.

### 선택 규칙

- 항상 중복 없는 3개를 제시한다.
- 현재 등급이 1 이상인 일반 능력이 하나라도 있으면 해당 능력과 연결되는 진화 유물을 최소 1개 포함한다.
- 나머지는 획득하지 않은 범용 유물과 다른 유효 진화 유물에서 결정한다.
- 유효 진화 유물이 없더라도 범용 3개로 선택지를 완성한다.
- 첫 보스의 `연쇄 탄두`와 두 번째 보스의 `연쇄 분열`은 함께 작동할 수 있다.
- 임시 구슬의 폭발과 추가 분열은 비재귀 규칙과 전역 상한으로 제한한다.

현재 코드의 임시 구슬 상한 `12`와 수명 `1500ms`는 이 작업에서 중앙 설정으로 옮긴다. 상위 유물 효과는 이 값을 직접 복사하지 않고 설정을 참조한다.

## 상태와 책임

`EncounterDirector`는 구간별 보스 일정을 읽어 보스 종류를 포함한 전환을 반환한다. 첫 구간에만 보스가 있다는 `section === 0` 분기는 제거한다.

`EnemyManager`는 분열형 부모와 파편을 별도 적 종류로 소유한다. 부모 사망 시 경계 clamp가 적용된 파편 사양을 생성하고, 활성 스프라이트 수와 별도로 활성 자리 비용을 스냅샷에 제공한다. 대형 결과도 실제 적 수와 함께 들어올 활성 자리 비용을 제공한다. `EncounterDirector`의 보충 생성 상한 검사는 현재 자리 비용과 들어올 자리 비용의 합을 사용한다.

첫 보스와 군체 보스는 작은 공통 제어 인터페이스를 사용한다.

- 시작
- 게임플레이 시간 갱신
- 일시정지와 재개
- 처치 여부와 피해 처리
- 적대 투사체와 예고 정리
- 패배·재시작 정리
- 자동 테스트용 읽기 전용 스냅샷

기존 `BossManager`는 첫 보스 전용으로 유지한다. 신규 `HiveBossManager`가 코어, 모듈, 전개 시계, 반사벽, 군체 투사체를 소유한다. 서로 다른 보스 규칙을 하나의 거대한 조건문으로 합치지 않는다.

`CombatScene`은 전환에 포함된 보스 종류로 올바른 제어기를 선택한다. 보스별 세부 상태를 직접 계산하지 않는다.

`BossBuild`와 보상 선택 규칙은 첫 보스 풀과 두 번째 상위 풀을 구분한다. 효과 계산은 구슬·피해 시스템이 필요한 값만 조회하도록 유지한다.

## 오류와 정리

- 잘못된 구간에서 보스 처치나 보상 재개를 호출하면 명확한 오류를 낸다.
- 보스 경고, 보스 전투, 보상 상태에서는 같은 구간 보스를 두 번 예약하지 않는다.
- 패배가 발생하면 군체 모듈, 반사벽, 탄환, 예고, 지연 폭발을 모두 제거한다.
- 보스 처치 시에도 같은 공격 객체를 먼저 제거한 뒤 보상 UI를 연다.
- 재시작은 구간, 보스 점수, 두 보스 객체, 두 단계 보상, 유물 효과 카운터를 모두 초기화한다.
- 일시정지 중 예약된 공격이나 잔향 폭발이 실시간 시계 때문에 먼저 실행되지 않게 게임플레이 경과 시간을 사용한다.

## 중앙 설정

다음 값은 `GAME_TUNING` 아래 한곳에서 정의하고 소비자가 참조한다.

- 구간별 보스 종류, 최소 시간, 목표 점수, 강제 시간, 경고 시간
- 위협 단계별 분열형 수와 종류별 활성 자리 비용
- 분열형 부모·파편 HP, 크기, 생성 간격, 경험치, 진행 점수, 돌파 피해
- 코어와 모듈 HP
- 결합, 예고, 노출 시간
- 사격 모듈 간격, 오프셋, 예고, 탄속, 피해
- 반사벽 크기, 이동 속도와 범위, 최소 통로, 재충돌 제한
- 코어 확산탄 각도, 간격, 탄속, 피해
- 영구·임시 구슬 전체 상한과 임시 구슬 수명
- 상위 유물의 횟수, 피해, 반경, 지연, 추가 구슬 수

설정 검증은 모든 숫자의 유한성, 양수·비음수·정수 조건, 시간 순서, 화면 내부 배치, 구슬과 탄환 상한을 검사한다.

## 자동 검증

### 순수 규칙과 단위 테스트

- 첫 보상 선택 전에는 두 번째 구간 시간이 흐르지 않는다.
- 보상 선택 완료 시 구간 시간, 점수, 생성 시계가 초기화된다.
- 첫 보상 뒤 위협 단계 2에서 시작하고 `60000ms`에 단계 3으로 전환한다.
- 두 번째 보상 뒤 세 번째 구간은 위협 단계 3에서 시작한다.
- 위협 단계 3은 `21~25`명, 활성 상한 `84`, 생성 간격 `5500ms`, 장갑·사수 각 `3`, 분열형 부모 `2`를 사용한다.
- 분열형 부모 사망은 파편 2개를 한 번만 만들고 다른 적과의 겹침을 허용한다.
- 화면 양쪽 경계의 부모도 파편을 월드 내부에 생성한다.
- 부모 자리 비용 `2`가 파편 두 개의 비용과 같아 분열 전후 활성 자리 합계가 유지된다.
- 부모는 보스 진행 점수 `2`, 파편은 `0`, 전체 경험치는 `3`으로 정산된다.
- 부모를 죽인 동일 피해 이벤트는 새 파편에 다시 적용되지 않는다.
- 두 번째 구간은 `150000ms + 110점` 또는 `210000ms`에서만 경고를 시작한다.
- 보스 전환이 올바른 보스 종류를 포함한다.
- 결합 `4000ms`, 예고 `1500ms`, 노출 `7000ms`의 직전과 경계에서 정확히 전이한다.
- 큰 프레임 간격에서도 한 갱신에서 상태를 건너뛰거나 공격을 몰아서 발사하지 않는다.
- 파괴된 모듈은 복귀·공격·반사하지 않는다.
- 모듈 전멸 시 코어가 영구 노출되고 확산탄은 유지된다.
- 반사벽은 구슬만 반사하고 같은 구슬 재타격 제한을 지킨다.
- 모든 군체 탄환이 전역 적대 탄환 상한을 공유한다.
- 상위 유물 선택은 3개, 무중복, 현재 빌드 연계 1개를 보장한다.
- 잔향 폭발과 연쇄 분열은 재귀하지 않는다.
- 임시 구슬 상한에서 추가 생성 수가 정확히 잘린다.
- 패배, 처치, 재시작이 모든 군체 객체와 상위 유물 상태를 정리한다.

### 브라우저 인수 테스트

가속 가능한 게임플레이 시계를 사용해 다음 전체 경로를 검증한다.

```text
첫 보스 진입
-> 첫 보스 처치
-> 첫 보상 선택
-> 두 번째 구간 시간과 점수 조건
-> 군체 보스 전개와 실제 물리 충돌
-> 두 번째 보스 처치
-> 상위 보상 선택
-> 세 번째 구간 재개
```

브라우저 테스트는 실제 영구·임시 구슬 충돌, 반사벽 반사, 플레이어 피격, 보상 UI 일시정지를 최소 한 번씩 통과해야 한다. 단순 디버그 함수 호출만으로 보스 피해와 처치를 증명하지 않는다.

## 플레이테스트

자동 검증 뒤 실제 플레이에서 다음을 기록한다.

- 첫 보상 선택부터 두 번째 보스 경고까지 걸린 시간
- 분열형이 숫자만 늘리는 적이 아니라 한 번 더 쓸어내는 재미로 읽히는지
- 벽 가까운 분열과 다수 적 겹침이 시각적으로 허용 가능한지
- 두 번째 보스 전투 시간과 남은 체력
- `7초` 노출 기회를 공 위치 때문에 반복해서 놓치는지
- 반사벽이 방해물과 유리한 튕김 도구로 모두 읽히는지
- 사격 모듈과 코어 확산탄을 동시에 피할 공간이 충분한지
- 모듈 우선 파괴와 코어 집중 공격이 둘 다 유효한지
- 두 번째 상위 유물 획득 직후 화력 상승이 체감되는지
- 임시 구슬 연쇄 조합이 화면 가독성과 성능 상한을 지키는지

플레이 결과에 따라 HP, 노출 시간, 사격 간격, 반사벽 이동 범위부터 조정한다. 구조 변경 없이 중앙 설정만으로 비교할 수 있어야 한다.

## 완료 기준

- 첫 보상 선택 뒤 두 번째 구간이 독립 시간과 점수로 진행된다.
- 첫 보스 이후 일반 적 위협이 이전 단계로 낮아지지 않는다.
- 위협 단계 3에서 분열형이 등장하고 분열 후에도 활성 상한과 월드 경계를 지킨다.
- 성과가 좋으면 `150초` 이후 두 번째 보스를 앞당기고, 늦어도 `210초`에는 진입한다.
- 군체 보스가 첫 보스와 다른 전개·집중 공격·모듈 우선순위 판단을 만든다.
- 공 회수 타이밍 때문에 한 번의 노출을 놓쳐도 반복 기회가 충분하다.
- 보스 처치 뒤 상위 유물 3개 중 하나를 선택하고 세 번째 구간이 즉시 이어진다.
- 모든 조정 수치가 중앙 설정에서 변경 가능하다.
- 전체 단위 테스트, 브라우저 테스트, 타입 검사, 프로덕션 빌드가 통과한다.
