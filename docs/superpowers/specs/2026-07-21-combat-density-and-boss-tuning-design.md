# 전투 밀도와 첫 미드보스 튜닝 설계

## 목표와 범위

첫 미드보스 수직 슬라이스의 플레이테스트 결과를 반영한다. 보스전은 현재 길이와 생존 압박을 유지하면서 큰 패턴 사이의 빈 시간을 단발 기본 공격으로 채운다. 보스는 더 크고 느리게 움직여 약점 조준을 쉽게 읽게 한다. 일반 적은 훨씬 천천히 내려오지만 더 많이, 더 단단하게 등장해 벽돌 무리가 쌓이고 연쇄적으로 쓸리는 감각을 강화한다.

이번 범위는 다음을 포함한다.

- 밸런스와 표현 수치의 중앙 tuning 구조
- 첫 미드보스 크기와 이동 속도 조정
- 큰 패턴 사이의 단발 조준 기본 공격
- 적대 탄환과 아군 분열 구슬의 시각적 구분
- 일반 적 하강 속도, 체력, 대형 크기, 최대 활성 수, 생성 해제 조건 조정
- 자동 검증과 수동 플레이테스트 항목 갱신

메인 구슬을 동료처럼 개별 특성화하고, 보유한 메인 구슬 조합에 따라 추가 기능을 만드는 시스템은 후속 설계로 남긴다. 이번 구현은 그 시스템을 추측해 확장하지 않는다.

## 단일 tuning 진입점

`src/game/config/gameTuning.ts`를 gameplay 및 전투 표현 수치의 단일 진입점으로 둔다. 설정은 하나의 평면 상수 목록이 아니라 다음 도메인으로 묶인 typed object다.

- `boss`: 본체 크기, 이동 속도, 약점 배치와 판정, 기본 공격과 큰 패턴 수치
- `enemies`: 종류별 체력과 하강 속도
- `encounter`: 초기 적 수, 단계별 대형 크기, 최대 활성 적 수, 생성 간격과 생성 해제 높이
- `projectiles`: 적대 투사체 속도, 피해, 발사 간격과 예고 시간
- `visual`: 아군 구슬과 적대 투사체의 색, 외곽선, 표시 크기

설정은 `as const satisfies GameTuning` 형태로 키 누락과 잘못된 값 형식을 컴파일 시점에 막는다. 물리 크기, 텍스처 크기, 이동 경계, 약점 위치처럼 같은 의미에서 파생되는 값은 원본 수치를 복사하지 않고 계산한다. 배열 인덱스, 부호, 수학 공식의 `0`과 `1`처럼 밸런스와 무관한 구현 상수까지 중앙 설정에 넣지는 않는다.

이번에 수정하는 기존 수치는 중앙 tuning으로 옮긴다. 관련 모듈과 테스트는 tuning을 참조하며 동일 의미의 숫자를 다시 선언하지 않는다.

## 첫 미드보스 크기와 이동

초기 tuning은 다음과 같다.

- 본체 표시 및 물리 크기: `168x96px`
- 약점 표시 크기: `18x48px`
- 약점 물리 판정: `22x52px`
- 약점과 본체 가장자리의 겹침: `5px`
- 코어 표시 및 판정 크기: 현재 `32px` 표시와 `28px` 판정 유지
- 수평 이동 최고 속도: `35px/s`
- 현재 약점과 코어 체력: 유지
- 현재 큰 패턴 단계, 피해, 예고 시간: 유지

약점 중심은 `(본체 폭 + 약점 표시 폭) / 2 - 겹침`만큼 중심에서 떨어진 좌우 대칭 위치로 계산한다. 화면 이동 경계는 본체만이 아니라 약점 물리 판정까지 포함한 보스 전체 충돌 폭에서 계산해 가장자리에서 약점이 잘리지 않게 한다. 보스 높이와 겹치는 일반 적의 장애물 구간 및 보스 이동 띠도 전체 충돌 크기에서 계산한다. 경계 부근 감속과 반전 규칙은 유지하되 최고 속도만 tuning에서 읽는다.

크기가 커져도 일반 적과 적 탄환은 보스보다 앞에 표시되는 기존 depth 규칙을 유지한다.

## 단발 조준 기본 공격

기본 공격은 큰 패턴 사이의 공백만 채운다. 큰 패턴과 동시에 겹쳐 회피 부담을 불명확하게 만들지 않는다.

- 발사 간격: `900ms`
- 투사체 수: `1`
- 조준: 발사 시점의 플레이어 위치를 향해 방향 고정
- 투사체 속도: `150px/s`
- 피해: `1`
- 발사 전 총구 점멸: `150ms`
- 바닥 또는 이동 경로 예고: 없음

큰 패턴 예고가 시작되면 기본 공격 시계는 정지한다. 이때 총구 점멸만 시작되고 아직 발사되지 않은 기본 공격이 있으면 점멸과 예약 발사를 취소한다. 큰 패턴의 투사체 또는 지원 사격이 실행되면 기본 공격 다음 실제 발사 시점을 실행 시점의 `900ms` 뒤로 다시 잡고, 그 `150ms` 전에 총구 점멸을 시작한다. 따라서 큰 패턴과 기본 공격이 같은 프레임 또는 바로 이어진 프레임에 발사되지 않는다. 레벨업, 보스 보상, 탭 비활성화 같은 기존 전투 pause 동안 두 공격 시계 모두 진행하지 않는다.

기본 공격은 기존 전체 적대 탄환 상한을 공유한다. 상한이 찼으면 해당 발사는 건너뛰며, 밀린 탄환을 이후 한꺼번에 발사하지 않는다. 보스 처치, 플레이어 패배, Scene 종료 시 총구 점멸과 예약 공격을 모두 정리한다.

## 적대 투사체 판독성

일반 사격 적의 탄환, 보스 기본탄, 보스 큰 패턴 탄환은 적대 계열임을 즉시 알 수 있도록 붉은색과 주황색 계열을 사용한다. 작은 화면과 색각 차이에서도 아군 분열 구슬과 구분되도록 어두운 중심 또는 외곽선을 함께 사용한다.

아군 메인 구슬과 임시 분열 구슬은 밝은 청색 계열을 유지한다. 적대/아군 구분은 색상 하나에만 의존하지 않고 밝기 대비와 외곽 형태도 다르게 한다. 초기 팔레트는 다음과 같다.

| 대상 | 채움 | 외곽/중심 | 형태 |
|---|---:|---:|---|
| 영구 구슬 | `0xffffff` | `0x4ddcff` | 밝은 원과 청색 외곽선 |
| 임시 분열 구슬 | `0x8cf7ff` | `0x167d9a` | 작은 밝은 원과 청록 외곽선 |
| 일반 적 탄환 | `0xff4d5a` | `0x4a0710` | 붉은 원과 어두운 중심 |
| 보스 기본/조준탄 | `0xff704d` | `0x4a0710` | 붉은 주황 원과 어두운 중심 |
| 보스 낙하물 | `0xff7b55` | `0x4a0710` | 세로로 긴 둥근 사각형 |

팔레트 값과 표시 크기는 `gameTuning.visual`에서 한 번만 정의하고 모든 텍스처 생성 경로가 이를 참조한다.

## 일반 적 속도, 체력, 밀도

초기 tuning은 다음과 같다.

| 항목 | 현재 | 변경 |
|---|---:|---:|
| 하강 속도 | `18px/s` | `8px/s` |
| 초기 적 수 | `20` | `26` |
| 기본 적 HP | `1` | `2` |
| 사격 적 HP | `1` | `2` |
| 장갑 적 HP | `3` | `5` |

단계별 대형 크기와 최대 활성 수는 다음처럼 바꾼다.

| 위협 단계 | 대형 크기 | 최대 활성 적 |
|---|---:|---:|
| 0 | `13~15` | `48` |
| 1 | `15~18` | `60` |
| 2 | `18~21` | `72` |

현재 생성 간격 `8000/7000/6000ms`는 첫 조정에서 유지한다. 다만 느려진 속도에서 기존 `PLAYER_MIN_Y=98`을 생성 해제 조건으로 재사용하면 실제 보충이 지나치게 늦어진다. 생성 전용 `reinforcementReleaseY=50`을 `gameTuning.encounter`에 추가한다.

새 대형은 다음 조건을 모두 만족할 때만 생성한다.

1. 단계별 생성 간격 경과
2. 기존 적의 최상단 위치가 `reinforcementReleaseY` 이상
3. 현재 적 수와 새 대형 적 수의 합이 최대 활성 수 이하

동적 목표 개체 수나 프레임별 자동 보충은 도입하지 않는다. 대형 seed와 bag 순서는 유지해 같은 run seed에서 결정론적으로 재현할 수 있어야 한다. 느린 이동 때문에 대형 사이가 겹치는 정도와 실제 화면 밀도는 수동 플레이테스트에서 확인한다.

## 보스 진입과 난이도 보존

첫 보스의 최소 진입 시간 `120초`, 처치 점수 목표 `70`, 강제 진입 `210초`는 우선 유지한다. 일반 적 체력이 올라 처치 속도가 낮아지지만 적 수와 화면 체류 시간도 함께 증가한다. 실제 플레이에서 점수 진입이 거의 사라지고 대부분 강제 진입만 발생하면 후속 튜닝에서 점수 목표만 독립적으로 낮춘다.

일반 적의 최하단 도달 피해, 일반/보스 적 탄환 피해, 플레이어 체력은 이번 범위에서 바꾸지 않는다. 보스 HP와 큰 패턴 수치도 유지해 새 기본 공격과 크기·이동 변화의 효과를 분리해 평가한다.

## 구성 요소와 데이터 흐름

- `gameTuning`: 모든 조절값을 제공하고 도메인별 타입을 보장한다.
- `bossRules`: 큰 패턴 선택과 기본 공격 가능 시점 및 다음 발사 시점을 결정한다.
- `BossManager`: tuning에서 보스 geometry를 만들고 총구 점멸, 조준 방향 고정, 탄환 생성과 정리를 담당한다.
- `formationRules`: tuning의 적 수, 종류별 HP, 속도로 결정론적 대형을 생성한다.
- `EncounterDirector`: tuning의 간격, 생성 해제 높이, 최대 활성 수를 적용한다.
- `CombatScene`: tuning의 geometry와 visual 값을 사용해 임시 텍스처를 생성한다.

보스 기본 공격은 일반 적 탄환과 기존 전체 적대 탄환 수 콜백을 공유한다. 새 manager나 전역 event bus는 만들지 않는다.

## 설정 검증과 예외 처리

자동 테스트에서 다음 불변식을 검사한다.

- 속도, 체력, 피해, 발사 간격, 표시 크기는 유한한 양수다.
- 대형 최소 수는 양수이고 최대 수보다 작거나 같다.
- 각 단계 최대 활성 수는 해당 단계 최대 대형 크기 이상이다.
- 보스 전체 충돌 폭은 게임 화면 폭보다 작고 이동 가능한 수평 구간이 남는다.
- 약점 표시와 판정은 본체 가장자리에 `5px` 겹치며, 이동 경계에서 화면 밖으로 나가지 않는다.
- 생성 해제 높이는 새 대형 시작 높이보다 크고 플레이어 최소 높이보다 작다.
- 적대 투사체와 아군 분열 구슬은 동일한 색과 윤곽 조합을 사용하지 않는다.

개발 중 잘못된 tuning은 테스트와 개발 모드 검증에서 즉시 실패하게 한다. 런타임 중 조용히 보정해 설정 오류를 숨기지 않는다.

## 자동 검증

단위 테스트는 다음을 검증한다.

- 보스 본체, 이동 경계, 약점 위치와 판정이 중앙 크기 설정에서 파생된다.
- 보스 최고 속도 `35px/s`와 기존 경계 감속·반전이 함께 동작한다.
- 기본 공격이 `900ms`마다 한 발을 현재 플레이어 위치로 조준한다.
- 큰 패턴 예고와 실행 중 기본 공격이 없고, 실행 후 다음 기본 공격까지 온전한 간격이 남는다.
- pause 동안 기본 공격과 큰 패턴 시계가 진행하지 않는다.
- 적대 탄환 상한에서 기본 공격이 누적 발사되지 않는다.
- 적 종류별 HP, `8px/s` 속도, 초기 적 수와 단계별 수량 범위가 tuning과 일치한다.
- 생성 간격, `reinforcementReleaseY`, 최대 활성 수가 모두 생성 gate에 적용된다.
- 같은 seed가 같은 대형을 만들고 기존 style 다양성 조건을 보존한다.
- 적대 탄환과 아군 분열 구슬의 visual 설정이 구분된다.

브라우저 acceptance는 다음을 검증한다.

- 큰 패턴 사이에 기본탄이 반복되고 큰 패턴 예고 중에는 발생하지 않는다.
- 기본탄이 발사 시점의 플레이어 위치로 향하고 피해 `1`을 준다.
- 보스의 실제 크기와 느린 이동이 적용된다.
- 초기 화면과 후속 대형에서 변경된 적 수, HP, 속도가 적용된다.
- 느린 적에서도 생성 gate가 풀리면 기존보다 많은 적이 화면에 공존한다.
- 기존 이동, 구슬 회수, 레벨업, 보스 처치, 유물 선택, 재시작 acceptance가 유지된다.

## 수동 플레이테스트

다음 항목을 기존 미드보스 플레이테스트 문서에 추가한다.

- 기본탄이 충분히 자주 나오지만 큰 패턴과 겹쳐 억울하지 않은가.
- 총구 점멸만으로 단발 발사를 읽을 수 있는가.
- 적대 탄환과 임시 분열 구슬을 전투 중 즉시 구별할 수 있는가.
- 큰 보스와 `35px/s` 이동이 묵직하면서도 지나치게 쉬워지지 않는가.
- `8px/s` 적 무리가 벽돌처럼 쌓이되 화면이 정체되지는 않는가.
- HP `2/2/5`에서 구슬이 적 사이를 연쇄적으로 쓸어내는 체감이 유지되는가.
- 화면 최대 `48/60/72`가 모바일에서 프레임 저하나 판독성 저하를 만들지 않는가.
- 보스가 여전히 주로 `120~210초` 사이에 진입하는가.

첫 플레이 후에는 중앙 tuning 값만 바꿔 재검증한다. 구조 변경은 실제 수치 조정만으로 목표 밀도와 난이도를 만들 수 없다는 증거가 있을 때만 검토한다.
