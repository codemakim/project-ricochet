# Project Ricochet 코어 리디자인

## 1. 문서 목적

이 문서는 Project Ricochet의 정식 기준 게임 디자인이다. 첫 패들 프로토타입의 플레이테스트 결과를 반영해 핵심 조작과 구슬 순환을 교체한다. 기존 패들 기반 설계는 결정 이력으로만 남기며 신규 개발은 이 문서를 따른다.

이번 리디자인의 첫 목표는 로그라이트 콘텐츠가 아니다. 자유 이동, 조준, 자동 발사, 리코셰 공격, 구슬 회수가 하나의 빠른 전투 순환을 만드는지 검증한다.

## 2. 첫 프로토타입에서 확인한 문제

첫 프로토타입은 좌우 패들로 공 하나를 받아치는 구조였다. 플레이테스트에서 다음 문제가 확인되었다.

- 적은 계속 내려오지만 공은 먼 곳에서 불확실하게 튕겨 공격 주기가 맞지 않았다.
- 플레이어 앞의 적을 의도적으로 조준하기 어려웠다.
- 기본 적도 여러 번 맞아야 해 적 대형을 거의 정리하지 못했다.
- 바닥에서도 공이 살아남아 패들로 받아칠 동기가 약했다.
- 적 속도와 수치만 낮추면 전투가 수동적인 기다림으로 바뀌었다.

문제의 원인은 난이도 수치가 아니라 공에 대한 통제권 부족이다. 패들 반사를 폐기하고 플레이어가 구슬을 조준 발사하고 직접 회수하는 구조로 교체한다.

## 3. 제품 방향

- 장르: 리코셰 액션 로그라이트 슈팅
- 주 플랫폼: 모바일 웹 브라우저
- 보조 플랫폼: 데스크톱 웹 브라우저
- 화면: 450x800 기준 세로 전투장
- 모바일 조작: 좌우 듀얼 스틱
- 데스크톱 조작: `WASD` 이동과 마우스 조준
- 목표 런 시간: 약 12분, 허용 범위 10~15분
- 주무기: 회수와 재발사를 반복하는 영구 구슬

공격 버튼은 없다. 플레이어는 이동과 조준에 집중하고 회수된 구슬은 현재 조준 방향으로 자동 재발사된다.

## 4. 핵심 플레이 루프

1. 캐릭터와 영구 구슬 3개로 시작한다.
2. 왼쪽 스틱으로 캐릭터를 이동하고 오른쪽 스틱으로 조준한다.
3. 조준을 처음 활성화하면 보관 구슬이 짧은 간격으로 발사된다.
4. 구슬은 벽과 적에 반사하며 공격한다.
5. 캐릭터가 구슬 근처로 이동하면 구슬을 흡수한다.
6. 흡수된 구슬은 충전되고 재발사 대기열에 들어간다.
7. 구슬은 현재 조준 방향으로 자동 재발사된다.
8. 플레이어는 적 탄환과 접촉을 피하면서 회수 위치와 다음 각도를 선택한다.
9. 적이 최하단을 돌파하거나 플레이어를 공격하면 체력이 감소한다.
10. 체력이 0이면 런이 끝난다.

핵심 판단은 다음과 같다.

- 지금 조준 방향을 유지할지 바꿀지
- 위험한 위치로 이동해 구슬을 빨리 회수할지
- 구슬이 바닥 귀환할 때까지 안전하게 기다릴지
- 약한 적 무리를 쓸지 장갑·사수형을 우선 노릴지

## 5. 플레이어 조작

### 5.1 이동

- 플레이어는 작은 캐릭터다. 패들 명칭과 넓은 가로 형태를 사용하지 않는다.
- 화면 상단 약 10%는 적 생성 안전 구역이며 캐릭터가 진입할 수 없다.
- 나머지 전투장에서 상하좌우로 자유 이동한다.
- 캐릭터는 전투장 밖으로 나갈 수 없다.
- 적과 접촉하면 피해를 받고 서로 짧게 밀려나 겹침을 해소한다.

### 5.2 모바일 입력

- 왼쪽 화면 영역의 가상 스틱으로 이동한다.
- 오른쪽 화면 영역의 가상 스틱으로 조준한다.
- 오른쪽 스틱을 누르는 동안 조준 방향이 실시간으로 바뀐다.
- 손을 놓으면 마지막 유효 조준 방향을 유지한다.
- 두 스틱은 화면을 가리는 고정 장식보다 터치 시작 위치에 나타나는 방식을 우선 검토한다.

### 5.3 데스크톱 입력

- `WASD`로 이동한다.
- 마우스 위치로 조준한다.
- 마지막 마우스 조준 방향은 커서 이동이 멈춰도 유지된다.

### 5.4 조준선

- 현재 조준 방향을 항상 표시한다.
- 점선은 첫 벽 충돌 지점까지 표시한다.
- 첫 벽 반사 뒤의 방향을 짧게 추가 표시한다.
- 두 번째 이후의 예상 반사는 표시하지 않는다.

## 6. 영구 구슬의 생명주기

영구 구슬은 다음 상태를 순환한다.

`stored -> queued -> active -> attracting 또는 floor-returning -> stored`

### 6.1 보관과 발사

- 기본 영구 구슬 수는 3개다.
- 런 중 최대 영구 구슬 수는 6개다.
- 첫 조준 활성화 전에는 구슬을 보관한다.
- 첫 조준 활성화 시 보관 구슬을 발사 대기열에 넣는다.
- 대기열은 기본 약 0.1초 간격으로 한 개씩 발사한다.
- 여러 구슬을 같은 순간 회수해도 겹쳐 발사하지 않고 대기열 순서를 따른다.
- 영구 구슬 최대 상태에서 추가 영구 구슬을 얻으면 구슬 공격력으로 전환한다.

### 6.2 근접 흡수

- 초기 흡수 반경은 50px다. 플레이테스트 조정 범위는 40~64px다.
- 구슬과 캐릭터 사이에 고정 지형이 없으면 흡수를 시작한다.
- 적과 적 탄환은 흡수 시야를 차단하지 않는다.
- 흡수를 시작한 구슬은 공격과 충돌 판정을 제거한다.
- 구슬은 약 0.1초 동안 캐릭터 쪽으로 빨려 들어온 뒤 보관된다.
- 한번 시작된 흡수는 캐릭터가 움직여도 취소하지 않는다.

### 6.3 바닥 귀환

- 영구 구슬이 최하단 벽에 닿으면 `homeOnBottomHit` 설정에 따라 처리한다.
- 기본값 `true`에서는 일반 반사를 하지 않고 바닥 귀환 상태로 바뀐다.
- 바닥 귀환 중에는 적, 탄환, 벽, 지형 충돌과 공격 판정을 모두 무시한다.
- 거리에 관계없이 현재 캐릭터 위치를 추적한다.
- 캐릭터에 도착하면 보관되고 정상적으로 충전·재발사된다.
- 바닥 귀환은 직접 회수 전용 보너스를 발동하지 않는다.
- `homeOnBottomHit: false`에서는 최하단에서도 일반 반사한다.

### 6.4 시간제 귀환

- 기본값은 `autoReturnAfterMs: null`이며 비활성이다.
- 바닥 귀환과 자유 이동만으로 회수 문제가 해결되지 않을 때만 숫자를 넣어 비교한다.
- 시간제 귀환이 켜지면 제한 시간이 지난 영구 구슬은 바닥 귀환과 같은 무충돌 상태로 돌아온다.

### 6.5 회수 원인

구슬 회수는 원인을 보존한다.

- `proximity`: 캐릭터가 직접 접근해 회수
- `floorRecall`: 최하단 충돌 뒤 강제 귀환
- `timeoutRecall`: 선택적 시간 제한으로 귀환

모든 회수는 구슬 충전을 회복한다. 보호막, 밀치기, 회수 폭발 같은 직접 회수 보너스는 `proximity`에서만 발동한다.

## 7. 구슬 충전과 피해

### 7.1 기본 충전

- 재발사된 영구 구슬은 충전 3회를 얻는다.
- 충전 상태의 직접 충돌 피해는 1.5배다.
- 적에게 직접 적중할 때만 충전을 1회 소모한다.
- 벽 반사는 충전을 소모하지 않는다.
- 충전이 끝나도 기본 공격력으로 계속 공격한다.
- 기본 충전은 속도를 변경하지 않는다. 속도 증가는 능력으로만 제공한다.

### 7.2 항상 유지되는 능력

구슬이 밖에 있는 동안 다음 능력은 충전 여부와 관계없이 유지된다.

- 런에서 올린 기본 공격력
- 주기적 주변 피해
- 충돌 스플래시
- 화상·독 같은 지속 피해
- 연쇄 번개
- 구슬 크기와 기본 관통

기본 충전 1.5배는 직접 충돌 피해에만 적용한다. 전기장, 도트, 스플래시는 전용 강화 능력이 있을 때만 충전 배율을 받는다.

### 7.3 적 충돌 반응

- 기본값 `passThroughOnKill: false`에서는 적이 죽어도 구슬이 반사된다.
- `passThroughOnKill: true`에서는 해당 타격으로 적이 죽으면 기존 방향을 유지한다.
- 적이 살아남으면 설정과 관계없이 반사한다.
- 관통 능력은 이 실험 설정보다 우선한다.
- 같은 구슬과 같은 적 조합에는 약 0.08초 재타격 제한을 둔다.
- 스플래시, 전기장, 도트 피해는 구슬의 방향을 바꾸거나 충전을 추가 소모하지 않는다.

## 8. 임시 구슬

- 분열 능력으로 생성하며 영구 구슬 제한과 별도로 관리한다.
- 회수하거나 영구 구슬로 바뀌지 않는다.
- 짧은 시간 동안 공격한 뒤 소멸한다.
- 최하단에 닿으면 귀환하지 않고 즉시 소멸한다.
- 기본 임시 구슬은 다시 분열하지 않는다.
- 진화 능력만 제한적인 2차 분열을 허용한다.
- 동시에 존재하는 임시 구슬 상한은 30개다.

## 9. 적 밀도와 공격 압박

### 9.1 적 수와 체력

- 초반 화면 적 수 목표는 10~15마리다.
- 중반은 20~30마리다.
- 후반 일반 구간은 30~45마리다.
- 대부분의 기본 적은 충전 구슬 1회에 파괴된다.
- 장갑 적은 충전 적중 2~3회가 필요하다.
- 엘리트와 보스 외에는 단순 체력 증가를 난이도의 주축으로 사용하지 않는다.
- 기본 적이 상단에서 최하단까지 내려오는 시간은 약 25~30초다.

### 9.2 대형

- 약한 적은 사각형, 벌집, 지그재그 같은 벽돌형 대형으로 진입한다.
- 대형 안에는 캐릭터와 구슬이 통과할 수 있는 틈을 둔다.
- 화면을 빈틈없이 가로막는 적 벽은 만들지 않는다.
- 장갑 적은 대형의 반사 기둥 역할을 한다.
- 사수형은 대형 안쪽에 배치해 우선 공격 가치를 만든다.

### 9.3 공격 관리

적 숫자와 동시에 공격하는 적 숫자를 분리한다.

- 초반 동시 공격자 상한은 2마리다.
- 중반은 3마리다.
- 후반은 4마리다.
- 공격 관리자가 가능한 사수형 중 발사자를 선택한다.
- 화면 적 탄환 수에도 별도 상한을 둔다.
- 모든 공격은 색, 모양, 또는 소리로 예고한다.
- 화면 밖 기습과 회피 불가능한 탄막은 허용하지 않는다.

## 10. 생존과 실패

- 캐릭터 기본 체력은 10이다.
- 일반 적 탄환 피해는 1이다.
- 적 직접 접촉 피해는 1이다.
- 강한 탄환과 미사일 피해는 2다.
- 일반 적 최하단 돌파 피해는 2다.
- 장갑·엘리트 돌파 피해는 3~5다.
- 피격 뒤 약 0.6초 무적 시간을 둔다.
- 보호막이 있으면 체력보다 먼저 소모한다.
- 적이 최하단에 도달하면 즉시 제거되고 돌파 피해를 준다.
- 체력 0이면 런이 실패한다.
- 첫 플레이 버전에는 부활이 없다.

## 11. 능력과 진화

### 11.1 슬롯

- 주 능력 5칸
- 보조 모듈 3칸
- 보스 유물 2칸
- 같은 능력은 최대 5등급
- 최대 등급, 대응 모듈, 호환 유물로 진화

### 11.2 주 능력

- 폭발: 모든 직접 적중에서 스플래시 피해
- 관통: 살아남은 적도 일정 횟수 통과
- 분열: 충전 적중 시 임시 구슬 생성
- 전기장: 구슬이 주기적으로 주변 적 공격
- 연쇄 번개: 직접 적중 시 주변 적으로 피해 전파
- 미사일: 구슬이 맞힌 적을 자동 추적
- 드론: 캐릭터 주변 공격 또는 탄환 요격

### 11.3 재발사와 충전 능력

- 과충전: 충전 횟수와 피해 증가
- 초탄 강화: 재발사 후 첫 적중 대폭 강화
- 운동 에너지: 충전 중 속도 증가
- 벽 가속: 첫 적중 전 벽 반사마다 피해 증가
- 연쇄 충전: 충전 상태 처치 시 충전 일부 회복
- 속사 장치: 재발사 간격 감소
- 동시 출격: 연속 발사되는 뒤 구슬 강화

### 11.4 회수 능력

- 흡수 반경과 흡수 속도 증가
- 직접 회수 시 보호막 획득
- 직접 회수 시 주변 적 밀치기
- 직접 회수한 구슬의 다음 첫 적중 강화
- 짧은 시간 안에 여러 구슬 직접 회수 시 보너스

### 11.5 진화 예시

분열 5등급, 충전 모듈, 증식 유물을 보유하면 직접 회수한 충전 구슬의 첫 적중에서 임시 구슬이 대량 생성된다. 임시 구슬은 적 무리 안에서 짧게 연쇄 반사하며 고밀도 대형을 무너뜨린다.

보스 유물 선택지에는 현재 빌드와 연결되는 항목을 최소 1개 보장한다.

## 12. 보스

### 12.1 중간 보스

- 화면 상단과 중앙을 천천히 이동한다.
- 큰 사각 몸체와 2~3개 약점을 가진다.
- 외장은 구슬을 반사하고 열린 약점은 큰 피해를 받는다.
- 미사일, 낙하물, 조준 사격 중 2개 패턴을 조합한다.
- 소수의 약한 적 대형을 소환해 연쇄 파괴 기회를 제공한다.
- 처치하면 보스 유물 3개 중 1개를 선택한다.

### 12.2 최종 보스

- 장갑 부위를 파괴해 핵을 노출한다.
- 구슬만 반사하는 임시 벽으로 궤적을 바꾼다.
- 약한 적 대형을 소환해 빌드의 광역 화력을 시험한다.
- 후반에는 이동 경로를 제한하는 탄막을 사용하되 캐릭터를 완전히 가두지 않는다.
- 바닥 귀환을 켠 경우 임시 벽으로 구슬이 고립되어도 복구할 수 있다.
- 처치하면 스테이지가 끝나고 콘텐츠를 해금한다.

## 13. 실험 설정과 비교 기준

첫 리디자인 프로토타입의 기본 설정은 다음과 같다.

```ts
const experiment = {
  passThroughOnKill: false,
  homeOnBottomHit: true,
  autoReturnAfterMs: null,
};
```

세 값은 플레이어가 런에서 얻는 능력이 아니라 개발용 비교 설정이다. 한 번의 비교에서는 한 값만 바꾸고 적 대형, 속도, 구슬 수는 동일하게 유지한다.

### 13.1 처치 후 진행 방향

`passThroughOnKill`만 바꾸고 같은 적 대형과 속도로 비교한다.

- 적 대형 정리 속도
- 구슬이 적 무리 안에 머무는 시간
- 구슬 회수 빈도
- 최하단 돌파 수
- 반사가 재밌는지 답답한지
- 관통 능력의 차별성

### 13.2 바닥 귀환

`homeOnBottomHit`만 바꾸고 비교한다.

- 직접 회수와 바닥 귀환 비율
- 구슬 없이 기다리는 시간
- 조준 수정 빈도
- 직접 회수의 성취감
- 바닥 귀환을 믿고 회수를 무시하는지
- 전투 흐름이 끊기는지

바닥 귀환이 켜져도 직접 회수는 더 빠르고 직접 회수 전용 능력을 발동하므로 가치가 있어야 한다.

## 16. 시스템 경계와 데이터 흐름

### 16.1 시스템

- 입력: 이동 벡터와 조준 벡터 생성
- 캐릭터: 이동, 충돌, 체력, 흡수 중심점
- 조준: 조준 방향 유지와 첫 반사 경로 계산
- 구슬: 상태, 충전, 충돌, 회수 원인
- 발사 대기열: 구슬을 일정 간격으로 현재 방향에 발사
- 적 대형: 배치와 하강
- 공격 관리: 동시 공격자와 탄환 수 제한
- 실험 설정: 세 비교 옵션 제공
- 표현: 상태를 시각화하되 전투 판정을 변경하지 않음

### 16.2 주요 흐름

`직접 회수 -> proximity 기록 -> 보관 -> 충전 -> 대기열 -> 현재 방향 재발사`

`최하단 충돌 -> floorRecall 기록 -> 충돌 제거 -> 캐릭터 추적 -> 보관 -> 충전 -> 재발사`

`적 직접 적중 -> 기본 피해와 항상 유지 효과 -> 충전 보너스 -> 충전 소모 -> 적 생존 여부와 설정으로 반사 결정`

`적 공격 요청 -> 공격 관리 허가 확인 -> 탄환 생성 -> 활성 공격자와 탄환 상한 갱신`

## 17. 안정성과 성능

- 브라우저가 백그라운드로 가면 자동 일시정지한다.
- 영구 구슬은 상태 전환 중 중복 흡수나 중복 대기열 등록을 허용하지 않는다.
- 귀환 상태 구슬은 공격과 물리 충돌을 모두 제거한다.
- 조준 벡터가 0이면 마지막 유효 방향을 유지한다.
- 발사 위치는 캐릭터 충돌 반경 밖으로 보정해 즉시 재흡수되지 않게 한다.
- 임시 구슬, 탄환, 이펙트는 객체 재사용 풀을 사용한다.
- 적, 탄환, 임시 구슬에 각각 활성 개수 상한을 둔다.
- 전투 계산과 표현을 분리한다.
- 일반 전투 60fps를 목표로 한다.

## 18. 검증 계획

### 18.1 자동 검증

- 2D 이동 경계와 상단 진입 제한
- 마지막 유효 조준 방향 유지
- 첫 벽 반사 조준선 계산
- 발사 대기열 순서와 간격
- 구슬 상태 전환과 중복 등록 방지
- 직접 흡수의 거리·시야 조건
- 바닥 귀환의 무충돌·무공격 상태
- 회수 원인별 보너스 발동 여부
- 충전 횟수, 직접 피해 배율, 항상 유지 효과
- 처치 후 반사 설정과 관통 우선순위
- 동시 공격자와 탄환 수 상한
- 체력, 무적 시간, 돌파 피해
- 데스크톱과 모바일 입력 변환

### 18.2 직접 검증

- 모바일 두 엄지 조작 피로도
- 캐릭터와 구슬을 동시에 추적할 수 있는지
- 원하는 적 대형을 조준해 공략할 수 있는지
- 약한 적 대형이 연쇄 반사로 무너지는지
- 직접 회수를 위해 자연스럽게 이동하는지
- 바닥 귀환을 켜도 직접 회수 가치가 남는지
- 모든 구슬이 밖에 있는 상태가 한 번에 3초를 넘을 때도 이동 또는 회수로 흐름을 되찾을 수 있는지
- 5분 동안 필연적으로 죽는다는 느낌이 없는지

### 18.3 리디자인 프로토타입 승인 기준

- 조준한 방향과 실제 구슬 진입 방향이 일치한다.
- 공격하지 않는 약한 적 20마리 시험 대형의 80% 이상을 첫 적이 최하단에 도달하기 전에 제거한다.
- 적이 최하단에 도달하기 전에 대응할 시간이 있다.
- 회수와 재발사가 빠르게 순환한다.
- 실험 설정을 바꾸면 차이를 즉시 비교할 수 있다.
- 모바일과 데스크톱 모두에서 조작이 이해 가능하다.
- 코어 루프가 승인되기 전에는 로그라이트 콘텐츠와 보스를 구현하지 않는다.
