# 레벨업과 전투 성장 설계

## 1. 목적

연속 적 유입으로 전투는 이어지지만 플레이 중 목표와 변화가 부족하다. 이번 수직 구현은 적 처치가 경험치로 이어지고, 레벨업 선택이 즉시 구슬의 성능과 화면 결과를 바꾸는 첫 성장 곡선을 만든다.

첫 구현은 네 가지 능력만 포함한다. 수치 성장 두 가지와 전투 방식이 변하는 폭발·분열을 함께 제공해 구현 범위를 통제하면서도 빌드 차이와 순간 화력을 확인한다.

## 2. 포함과 제외

### 2.1 포함

- 적 처치 경험치 자동 획득
- 레벨, 다음 레벨 경험치, 경험치 바
- 전투 완전 정지형 레벨업 선택
- 1~3개의 중복 없는 선택 카드
- 화력 증폭, 운동 에너지, 폭발, 분열
- 같은 능력 최대 5등급
- 임시 구슬 최대 12개
- 모바일 탭, 데스크톱 클릭과 숫자 키 선택
- visibility 정지와 레벨업 정지의 안전한 조합

### 2.2 제외

- 경험치 아이템과 회수
- 주 능력 슬롯 제한과 교체 UI
- 관통, 전기장, 연쇄 번개, 미사일, 드론
- 능력 진화와 보조 모듈
- 보스 유물
- 적 난이도의 플레이어 레벨 연동
- 정식 그래픽과 사운드

## 3. 경험치와 레벨

적이 플레이어 공격으로 죽는 순간 경험치를 자동 획득한다.

- 기본 적: `1 XP`
- 사수형: `2 XP`
- 장갑형: `3 XP`
- 최하단 돌파로 제거된 적: `0 XP`

런 시작 레벨은 0이며 첫 필요 경험치는 8이다. 다음 등급에 필요한 경험치는 다음 식을 사용한다.

```ts
xpRequiredForLevel(level: number): number {
  return 8 + level * 4;
}
```

따라서 레벨 1~5 도달 비용은 `8, 12, 16, 20, 24 XP`다. 첫 레벨업은 평균 플레이 15~20초를 목표로 한다.

획득 경험치는 초과분을 보존한다. 한 번의 정산으로 여러 레벨 조건을 만족하면 `pendingChoices`를 해당 횟수만큼 올린다. 모든 선택이 끝날 때까지 전투를 재개하지 않는다.

네 능력이 모두 5등급이면 레벨 20이다. 이 수직 구현에서는 이후 경험치 획득을 중단하고 레벨업 화면을 더 열지 않는다. 전체 능력 풀이 추가되면 이 제한을 제거한다.

## 4. 선택지 생성

- 선택지는 현재 5등급 미만인 능력만 사용한다.
- 같은 화면에 같은 능력을 두 번 표시하지 않는다.
- 가능한 능력이 3개 이상이면 3개를 표시한다.
- 가능한 능력이 1~2개면 남은 능력만 표시한다.
- 첫 레벨업에는 폭발 또는 분열을 최소 하나 포함한다.
- 각 카드에는 이름, 현재 등급, 선택 후 등급, 다음 등급 효과를 표시한다.
- 선택 순서는 런 seed를 사용하는 결정적 셔플로 만든다.
- 테스트는 고정 seed를 주입해 같은 선택지를 재현한다.

선택 완료 시 등급을 먼저 적용한 뒤 `pendingChoices`를 하나 줄인다. 대기 선택이 남았으면 다음 선택지를 즉시 표시한다. 없으면 레벨업 정지 이유를 해제한다.

## 5. 능력 규칙

### 5.1 화력 증폭

모든 직접 충돌 피해에 등급당 `+0.25`를 더한다.

| 등급 | 직접 피해 보너스 |
|---:|---:|
| 0 | 0 |
| 1 | 0.25 |
| 2 | 0.50 |
| 3 | 0.75 |
| 4 | 1.00 |
| 5 | 1.25 |

- 영구 구슬의 기본 직접 피해 `1.0`과 충전 직접 피해 `1.5`에 적용한다.
- 임시 구슬의 기본 직접 피해 `0.5`에도 적용한다.
- 충전 횟수를 모두 사용한 영구 구슬에도 유지된다.
- 폭발 피해에는 적용하지 않는다. 폭발은 자체 등급 수치를 사용한다.

### 5.2 운동 에너지

충전 횟수가 1회 이상 남은 영구 구슬의 속도를 등급당 10% 높인다.

```ts
chargedSpeed = 400 * (1 + rank * 0.1)
```

- 1~5등급 속도는 `440, 480, 520, 560, 600px/s`다.
- 충전 횟수가 0이 되는 직접 적중 직후 속도를 `400px/s`로 정규화한다.
- 회수 후 충전이 복원되면 다음 재발사부터 강화 속도를 사용한다.
- 능력 선택 순간 화면에 나가 있는 충전 영구 구슬도 현재 진행 방향을 유지한 채 새 속도로 정규화한다.
- 임시 구슬은 충전 상태가 아니므로 운동 에너지를 받지 않는다.

### 5.3 폭발

폭발 등급이 1 이상이면 영구·임시 구슬의 직접 적중 위치에 범위 피해를 만든다.

| 등급 | 반경 | 범위 피해 |
|---:|---:|---:|
| 1 | 48px | 0.50 |
| 2 | 56px | 0.75 |
| 3 | 64px | 1.00 |
| 4 | 72px | 1.25 |
| 5 | 80px | 1.50 |

- 직접 적중한 주 대상은 범위 피해에서 제외한다.
- 범위 안의 다른 활성 적에게 한 번씩 피해를 준다.
- 폭발로 죽은 적도 정상 경험치를 준다.
- 폭발 피해는 폭발, 분열, 충전 소모, 반사 판정을 다시 일으키지 않는다.
- 같은 직접 적중에서 폭발은 한 번만 발생한다.

### 5.4 분열

분열 등급이 1 이상인 영구 구슬이 충전 직접 적중에 성공하면 임시 구슬을 만든다.

| 등급 | 생성 수 |
|---:|---:|
| 1 | 1 |
| 2 | 1 |
| 3 | 2 |
| 4 | 2 |
| 5 | 3 |

- 임시 구슬 수명은 생성 후 `1,500ms`다.
- 기본 직접 피해는 `0.5`이며 화력 증폭을 적용한다.
- 기본 속도는 `440px/s`다.
- 1개 생성 시 좌우 편향을 피하도록 발동 순서마다 `+25도`, `-25도`를 번갈아 사용한다.
- 2개 생성 시 기존 진행 방향 기준 `-25도`, `+25도`로 발사한다.
- 3개 생성 시 `-30도`, `0도`, `+30도`로 발사한다.
- 벽과 적에 반사하며 같은 임시 구슬·적 조합의 재타격 제한은 `80ms`다.
- 폭발 능력은 적용하지만 분열을 다시 발동하지 않는다.
- 충전, 회수, 바닥 귀환, 재발사 대기열에 참여하지 않는다.
- 화면 활성 임시 구슬 상한은 12개다. 남은 자리가 부족하면 가능한 수만 생성하며 기존 구슬을 제거하지 않는다.
- 수명 종료, 장면 종료, 패배 시 즉시 제거한다.

## 6. 정지와 상태 우선순위

`CombatPauseController`는 다음 정지 이유를 독립적으로 관리한다.

- `visibility`
- `levelUp`
- `defeated`

정지 이유가 하나라도 있으면 게임플레이 업데이트, 물리, 구슬, 적, 탄환, 적 생성 시계, 전투 구간 시계를 진행하지 않는다. UI 렌더링과 레벨업 카드 입력은 동작한다.

- 레벨업 시작 시 구슬 위치, 속도, 충전, 조준 방향을 보존한다.
- 레벨업 선택이 끝나도 탭이 숨겨져 있으면 재개하지 않는다.
- 숨김 상태가 끝나도 레벨업 선택이 남아 있으면 재개하지 않는다.
- 숨김 시간이나 정지 시간을 재개 첫 프레임 `delta`에 포함하지 않는다.
- 사망과 레벨업이 같은 프레임에 확정되면 `defeated`가 우선한다.
- 패배 시 열린 레벨업 UI를 닫고 선택 입력을 받지 않는다.
- 재시작 시 모든 정지 이유, 경험치, 능력 등급, 임시 구슬을 초기화한다.

## 7. 구성 요소와 데이터 흐름

### 7.1 `ProgressionManager`

- 경험치, 레벨, 다음 필요 경험치, `pendingChoices`를 소유한다.
- 적 처치 경험치를 정산한다.
- 선택지 생성과 선택 소비를 조정한다.
- Phaser 객체나 적·구슬 구현을 알지 않는다.

### 7.2 `BuildState`

- 네 능력 등급을 소유한다.
- 직접 피해 보너스, 충전 속도, 폭발 수치, 분열 수를 계산한다.
- 5등급 제한과 선택 적용을 검증한다.

### 7.3 `LevelUpOverlay`

- 능력 카드 1~3개와 경험치·레벨 표시를 담당한다.
- 모바일 탭, 데스크톱 클릭, 숫자 `1~3` 입력을 같은 선택 명령으로 변환한다.
- 전투 계산과 정지 상태를 직접 변경하지 않고 선택 callback만 호출한다.

### 7.4 `CombatPauseController`

- 정지 이유 집합과 재개 가능 여부를 소유한다.
- 물리·scene time 정지와 재개를 한곳에서 수행한다.
- 기존 visibility 전용 boolean을 대체한다.

### 7.5 `TemporaryOrbManager`

- 임시 구슬 group, 수명, 반사, 적별 재타격 제한, 12개 상한을 소유한다.
- 영구 구슬의 회수와 발사 queue에 접근하지 않는다.

### 7.6 `EnemyManager`

- 플레이어 공격으로 적이 죽으면 `kind`, `enemyId`, `position`을 한 번 알린다.
- 범위 피해를 받아 각 적에게 한 번 적용한다.
- 돌파 제거는 처치 callback을 내지 않는다.
- 경험치, 능력 등급, UI를 알지 않는다.

### 7.7 전투 데이터 흐름

```text
직접 적중
-> 직접 피해와 충전 소모 계산
-> 대상 피해 적용
-> 폭발 적용
-> 조건을 만족하면 임시 구슬 생성
-> 각 사망을 한 번씩 처치 callback
-> 경험치 정산
-> pendingChoices 증가 시 levelUp 정지와 UI 표시
```

폭발과 분열은 직접 적중 event에서만 시작한다. 범위 피해 event와 임시 구슬의 분열 event를 만들지 않아 재귀를 구조적으로 차단한다.

## 8. 난이도 정책

이번 구현에서 현재 적 체력, 하강 속도, 생성 간격, 활성 상한을 바꾸지 않는다.

- 플레이어 레벨을 읽어 적을 즉시 강화하지 않는다.
- 적 난이도는 전투 시간과 구간 번호로만 상승한다.
- 성장 구현 후 직접 플레이에서 1~2분, 2~3분의 정리 속도와 생존 압박을 측정한다.
- 화력이 너무 앞서면 하강 속도보다 적 밀도, 장갑 비율, 사수 비율을 먼저 올린다.
- 강화 직후 적도 같은 비율로 강해지는 rubber-banding은 사용하지 않는다.

## 9. 실패 처리와 경계 조건

- 같은 적의 사망 callback은 직접·폭발·임시 구슬 피해가 겹쳐도 한 번만 발생한다.
- 5등급 능력은 선택지에서 제외한다.
- 가능한 선택지가 없으면 경험치 획득을 중단하고 UI를 열지 않는다.
- 선택 index가 현재 카드 범위를 벗어나면 무시한다.
- 이미 소비한 선택 입력은 다시 적용하지 않는다.
- 장면 종료 시 키 입력, 카드 입력, 타이머, 임시 구슬 collider를 모두 정리한다.
- 임시 구슬 상한에 도달해 일부가 생성되지 않아도 발동 순번은 한 번 진행한다.

## 10. 검증 기준

### 10.1 단위 테스트

- 적 종류별 경험치와 `8, 12, 16, 20, 24` 요구량
- 초과 경험치 보존과 여러 pending 선택
- 첫 선택의 폭발·분열 보장, 중복 없음, 5등급 제외
- 각 능력 1~5등급 수치 표
- 충전 소진 시 속도 복귀
- 폭발 주 대상 제외와 비재귀 피해
- 분열 각도, 수명, 피해, 80ms 재타격 제한, 12개 상한
- 정지 이유가 겹칠 때 마지막 이유가 해제될 때만 재개

### 10.2 브라우저 자동 테스트

- 8 XP 획득 시 적·탄환·구슬·전투 시계 정지
- 모바일 카드 탭과 데스크톱 클릭·숫자 선택
- 선택 후 등급과 UI가 갱신되고 전투 재개
- 폭발 선택 뒤 한 직접 적중으로 주변 적 피해
- 분열 선택 뒤 임시 구슬 생성과 1.5초 뒤 제거
- hidden과 레벨업이 겹쳐도 잘못 재개되지 않음
- 패배와 재시작 시 성장 상태 초기화
- production bundle에 개발 mutation helper가 노출되지 않음

### 10.3 직접 플레이

- 첫 레벨업이 평균 15~20초 안에 발생한다.
- 첫 능력 선택부터 전투 결과가 달라진다.
- 분열과 폭발 조합이 짧은 고밀도 화력 순간을 만든다.
- 임시 구슬이 12개여도 화면 가독성과 프레임이 유지된다.
- 레벨업 정지가 전투 흐름을 끊는 방해보다 짧은 보상으로 느껴진다.
- 1~3분 사이 성장 속도와 적 압박이 함께 올라간다.

## 11. 구현 순서

1. 경험치·레벨·선택지 순수 규칙
2. `BuildState`와 네 능력 수치
3. 처치 callback과 경험치 연결
4. 정지 이유 통합과 레벨업 UI
5. 화력 증폭과 운동 에너지
6. 폭발 범위 피해
7. 임시 구슬과 분열
8. 브라우저 자동 검증과 직접 플레이 기록
