# 비정형 적 대형과 밀도 확장 설계

## 목표

현재 적 배치가 일정한 격자와 교차 무늬를 반복해 인공적으로 보이는 문제를 줄인다. 붙어 있는 적 무리와 넓은 빈 공간을 함께 사용하고, 드물게 기존 격자형도 섞어 한 런 안에서 대형이 쉽게 익숙해지지 않게 한다.

이번 작업은 적 배치와 활성 수만 다룬다. 적 하강 속도, 체력, 경험치, 보스, 분열 구슬의 효과 상속은 변경하지 않는다.

## 초기 대형

- 초기 적 수는 `20`마리를 유지한다.
- 기존 교차 격자형으로 시작하지 않는다.
- 비정형 생성 규칙 4종 중 하나로 매번 새 대형을 만든다.
- 생성 규칙은 요청받은 적 수에 맞춰 행 수를 정하므로 초기 전용 좌표표를 갖지 않는다.
- 결과에는 인접한 적 무리와 2칸 이상의 빈 구간이 함께 나타나야 한다.
- 새 런과 재시작은 새 run seed를 사용하므로 시작 대형이 달라질 수 있다.

## 재사용 생성 규칙

모든 스테이지와 위협 단계가 같은 생성 규칙을 재사용한다.

- `cluster`: 두세 개의 씨앗에서 이웃 칸으로 자라 붙은 덩어리를 만든다.
- `pockets`: 넓게 채운 후보 영역에서 불규칙한 빈 구역을 남긴다.
- `bands`: 길이가 다른 짧은 가로 띠를 엇갈려 배치한다.
- `scatter`: 완전 균등 산개를 피하면서 작은 무리와 고립 적을 섞는다.
- `grid`: 기존 교차 격자의 성격을 유지하되 낮은 빈도로만 사용한다.

생성기는 `style + count + seed`를 입력받아 완성된 좌표를 반환한다. 고정 대형 좌표 목록이나 단계별 패턴 복사본을 두지 않는다. 적 수가 바뀌거나 새 스테이지가 추가되어도 생성 규칙은 수정하지 않고 단계 설정만 바꾼다.

## 보충 대형 크기

대형 크기는 고정하지 않고 단계별 범위를 사용한다.

- 위협 단계 0: `9~11`마리
- 위협 단계 1: `11~13`마리
- 위협 단계 2: `13~15`마리

생성기는 기존 8개 열을 사용하고 요청 수에 따라 필요한 행 수를 계산한다. 시드로 좌우 치우침과 행별 `-10`, `0`, `+10px` 오프셋을 정한다. 모든 적 중심은 기존 안전 범위 `36~414px` 안에 남아야 하며 같은 셀을 중복 사용하지 않는다.

## 선택 규칙

단순 독립 난수 대신 run seed와 생성 순번으로 결정되는 셔플 백을 사용한다.

- 비정형 생성 규칙 4종을 각각 두 번 넣는다.
- 격자형 대형을 한 번 넣는다.
- 한 백은 총 9개이며 격자형 비율은 `1/9`이다.
- 같은 생성 규칙은 연속해서 나오지 않는다. 백 경계에서도 이전 규칙과 다음 규칙이 같으면 첫 항목을 교환한다.
- 선택된 규칙은 현재 단계의 적 수 범위에서 수를 뽑고 별도 파생 seed로 좌표를 생성한다.
- 같은 run seed와 생성 순번은 항상 같은 결과를 만든다.
- 다른 run seed는 초기 대형 또는 첫 9개 보충 순서 중 하나 이상이 달라야 한다.

run seed 생성은 `CombatScene`이 담당한다. 순수 대형 규칙은 seed를 입력으로만 받고 직접 난수를 읽지 않는다. 자동 테스트는 고정 seed를 주입해 재현한다.

## 밀도

활성 적 상한은 다음과 같이 올린다.

- 위협 단계 0: `32`
- 위협 단계 1: `40`
- 위협 단계 2: `48`

다음 값은 유지한다.

- 적 하강 속도 `18px/s`
- 보충 간격 `8/7/6초`
- 상단 공간 확보 조건
- 동시 사수 2명, 적 탄환 12개 상한
- 적 체력과 적 종류별 경험치

보충 대형의 실제 크기로 활성 상한을 검사한다. 남은 칸이 대형 전체보다 적으면 대형을 잘라 넣지 않고 기존처럼 대기한다.

## 구성 경계

- `formationRules`는 재사용 가능한 순수 좌표 생성기, 선택 규칙, 초기·보충 대형 생성을 담당한다.
- `EncounterDirector`는 run seed와 생성 순번을 보관하고 현재 단계의 다음 대형을 요청한다.
- `CombatScene`은 런 시작 시 seed를 만들고 초기 대형과 `EncounterDirector`에 전달한다.
- `EnemyManager`는 완성된 `EnemySpec[]`만 받아 생성하며 선택 규칙을 알지 않는다.

현재 인터페이스를 크게 바꾸지 않고 seed 입력만 추가한다. 생성 규칙은 현재 8열 전장에만 맞추며 범용 레벨 편집기나 설정 UI는 만들지 않는다.

## 검증

단위 테스트:

- 여러 seed의 초기 대형이 정확히 20마리이며 격자형이 아니다.
- 각 생성 규칙은 요청 수와 같은 수의 중복 없는 안전 범위 좌표를 만든다.
- 여러 seed에서 같은 생성 규칙도 서로 다른 배치를 만든다.
- 단계별 결과가 `9~11`, `11~13`, `13~15` 범위에 있다.
- 한 셔플 백 9개에서 격자형은 정확히 1회다.
- 같은 생성 규칙이 연속하지 않는다.
- 좌우 반전과 행 오프셋 뒤에도 위치가 안전 범위 안이다.
- 같은 seed는 같은 결과, 다른 seed는 다른 결과를 만든다.
- 활성 상한은 `32/40/48`, 간격은 `8/7/6초`다.
- 상한이 부족하면 보충 대형 전체가 대기한다.

브라우저 자동 테스트:

- 초기 대형이 20마리다.
- 원래 적이 남아 있어도 비정형 보충 대형이 추가된다.
- 한 런에서 연속 생성된 대형의 배치가 동일하지 않다.
- 사수와 탄환 상한, pause, restart가 유지된다.

사용자 플레이:

- 시작 화면이 교차 격자로 보이지 않는지
- 붙은 적 무리와 큰 빈 공간이 함께 느껴지는지
- 반복 패턴을 빠르게 예측하게 되지 않는지
- 늘어난 적 수가 화면 가독성과 성능을 해치지 않는지

## 다음 독립 작업

분열 구슬은 현재 폭발을 포함한 메인 구슬 효과를 상속한다. 이 상속을 기본 분열에서 제거하고 보스 유물로 이동하는 설계는 적 대형 작업 완료 뒤 별도 기능 작업으로 진행한다.
