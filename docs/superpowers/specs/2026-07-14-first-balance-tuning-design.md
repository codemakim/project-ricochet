# 1차 성장·적 밀도 밸런스 조정

## 목표

첫 레벨업이 너무 빠르고 적 밀도가 낮다는 사용자 플레이 결과를 반영한다. 적은 더 느리게 내려오지만 화면에는 더 많이 남게 만들고, 늘어난 처치 수가 레벨업을 다시 앞당기지 않도록 경험치 곡선을 함께 늦춘다.

이번 작업은 현재 코어 전투의 1차 수치 조정이다. 세부 밸런스 확정이나 적 레벨 시스템은 포함하지 않는다.

## 수치 변경

### 레벨 경험치

- 현재 공식: `8 + level * 4`
- 변경 공식: `12 + level * 5`
- 레벨 1~5 필요 경험치: `12, 17, 22, 27, 32`
- 경험치 초과분, 복수 레벨업 대기, 최대 랭크 규칙은 유지한다.

### 적 하강 속도

- 초기 대형과 모든 보충 대형: `22px/s`에서 `18px/s`로 변경한다.
- 구간에 따른 추가 속도 증가는 이번 작업에 포함하지 않는다.

### 보충 대형 크기

- 위협 단계 0: `6`마리에서 `8`마리
- 위협 단계 1: `8`마리에서 `10`마리
- 위협 단계 2: `10`마리에서 `12`마리
- 기본·장갑·사수 적을 기존 2행 구조 안에서 섞는다.
- 사수형 비율과 동시 사수·탄환 상한은 기존 안전장치를 유지한다.

### 활성 적 상한

- 위협 단계 0: `22`에서 `28`
- 위협 단계 1: `26`에서 `34`
- 위협 단계 2: `30`에서 `40`

### 유지하는 값

- 보충 생성 간격: `8초 / 7초 / 6초`
- 상단 공간 확보 조건
- 적 체력과 적 종류별 경험치
- 플레이어 체력, 구슬 성능, 능력 수치

생성 간격은 유지해 대형 크기·상한·속도 변경 효과를 먼저 분리해서 확인한다.

## 기대 결과

- 첫 레벨업은 기존보다 늦어진다.
- 적 하강 압박은 완만해지지만 화면 적 수는 늘어난다.
- 구슬이 다수 적 사이를 연속으로 튕기며 정리하는 시간이 늘어난다.
- 성장 체감은 유지하되 보스가 없는 현재 구간에서 과도하게 빨리 강해지는 현상을 줄인다.

## 검증

자동 테스트는 다음을 확인한다.

- 경험치 공식의 정확한 첫 5개 값
- 초기·보충 대형의 `18px/s`
- 단계별 보충 대형 크기 `8/10/12`
- 단계별 활성 적 상한 `28/34/40`
- 보충 생성 간격과 상단 공간 조건이 유지됨
- 기존 unit, desktop/mobile E2E, production build 회귀 없음

사용자 플레이에서 다음을 다시 확인한다.

- 첫 레벨업 도달 시간
- 적이 느리지만 충분히 많이 보이는지
- 1~3분 성장 속도와 적 압박
- 폭발·분열의 다수 적 정리감

## 제외 범위

- 적 레벨과 구간별 체력 성장
- 보스와 보스 유물
- 능력 수치 재조정
- 그래픽·사운드 제작
- 모바일 실기 검증
