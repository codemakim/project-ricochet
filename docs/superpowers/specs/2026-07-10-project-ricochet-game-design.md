# Project Ricochet 게임 디자인

## 1. 문서 목적

Project Ricochet은 브레이크아웃의 반사 조작과 로그라이트 슈팅의 성장 구조를 결합한 세로형 액션 게임이다. 이 문서는 전체 제품 방향과 첫 플레이 가능한 버전의 범위를 정의한다.

첫 개발 목표는 콘텐츠 양이 아니다. 12분짜리 한 판에서 다음 세 가지가 재미있는지 검증한다.

1. 공을 패들로 직접 받아쳐 강화하는 행동
2. 적 탄환을 피하면서 하강하는 적을 처리하는 압박
3. 능력 선택으로 후반 화력이 폭발적으로 성장하는 경험

수치와 콘텐츠 양은 플레이테스트로 바꿀 수 있다. 공이 바닥에서도 살아남고, 패들 타격이 공을 강화하며, 패들 체력이 0이 되면 런이 끝난다는 핵심 규칙은 유지한다.

## 2. 제품 방향

### 2.1 장르와 플랫폼

- 장르: 리코셰 액션 로그라이트 슈팅
- 주 플랫폼: 모바일 웹 브라우저
- 보조 플랫폼: 데스크톱 웹 브라우저
- 화면: 세로 고정 전투 영역
- 조작: 한 손 좌우 드래그
- 목표 플레이 시간: 한 런 약 12분, 허용 범위 10~15분

### 2.2 핵심 정체성

공이 주무기다. 자동 미사일, 레이저, 드론은 공을 대체하지 않고 공 빌드를 보완한다. 모든 패들은 같은 조작을 사용하지만 시작 특성과 전용 능력으로 빌드 출발점이 달라진다.

플레이어의 주된 판단은 다음과 같다.

- 어느 위치에서 공을 받아 반사각을 바꿀지
- 공을 받기 위해 위험한 위치로 이동할지
- 적 탄환과 최하단 돌파 위험 중 무엇을 먼저 피할지
- 새 능력을 열지, 기존 능력을 강화해 진화에 가까워질지

## 3. 핵심 플레이 루프

1. 패들 1개와 기본 공 1개로 런을 시작한다.
2. 패들을 좌우로 움직여 탄환을 피하고 공을 받아친다.
3. 패들에 닿은 공은 충전되어 정해진 횟수만큼 강화 공격을 한다.
4. 적 처치 경험치는 자동 획득한다.
5. 레벨업하면 전투가 멈추고 능력 3개 중 1개를 선택한다.
6. 회복, 보호막, 보물은 아래로 떨어지며 패들로 받아야 획득한다.
7. 적이 최하단에 도달하면 해당 적은 제거되고 패들이 큰 피해를 받는다.
8. 패들 체력이 0이 되면 런이 실패한다.
9. 12분의 최종 보스를 처치하면 스테이지를 클리어한다.

### 3.1 한 런의 진행

- 0~4분: 기본 빌드 형성
- 4분: 첫 중간 보스와 보스 유물 선택
- 4~8분: 적 밀도와 투사체 증가
- 8분: 강화 중간 보스와 두 번째 보스 유물 선택
- 8~12분: 빌드 완성, 엘리트 조합 출현
- 12분: 스테이지 최종 보스

최종 보스 보상은 영구 능력치가 아니라 새 패들, 능력, 유물, 스테이지, 위협도 같은 콘텐츠를 해금한다.

## 4. 전투 규칙

### 4.1 공과 반사

- 공은 벽, 천장, 바닥, 적에 반사한다.
- 공은 바닥에 닿아도 사라지지 않는다.
- 지나치게 수평인 궤적은 최소 수직각으로 보정한다.
- 능력으로 공 속도가 오를 수 있지만 조작 가능한 최대 속도를 둔다.
- 패들 중앙 정타는 충전량과 피해 보너스에 유리하다.
- 패들 가장자리 타격은 큰 반사각 변경에 유리하다.
- 영구 공은 런 동안 유지하며 기본 최대 6개다.
- 영구 공이 최대일 때 추가 공 획득은 공 공격력 증가로 바뀐다.

### 4.2 충전

- 패들에 닿은 공은 기본 3회의 충전 횟수를 얻는다.
- 충전된 공은 피해와 연결 능력의 효과가 강해진다.
- 바닥 반사는 공을 살리지만 충전을 주지 않는다.
- 시간제 가속과 과충전은 기본 규칙이 아니라 선택 능력으로 제공한다.
- 멀티볼의 각 공은 충전 상태를 따로 가진다.

### 4.3 패들과 생존

- 패들 기본 체력은 10이다.
- 일반 탄환 피해는 1이다.
- 강한 탄환과 미사일 피해는 2다.
- 일반 적의 최하단 도달 피해는 2다.
- 장갑형과 엘리트의 최하단 도달 피해는 3~5다.
- 피격 후 짧은 무적 시간으로 겹친 탄환의 연속 피해를 막는다.
- 보호막이 있으면 체력보다 먼저 소모한다.
- 첫 플레이 버전에는 부활이 없다.

수치는 조정 대상이지만 최하단 돌파 피해가 일반 탄환 피해보다 크다는 관계는 유지한다. 적을 일부러 흘려보내는 전략이 회피보다 유리해져서는 안 된다.

### 4.4 적 탄환

- 기본 공은 적 탄환과 충돌하지 않는다.
- 요격, 반사, 흡수 능력을 얻으면 공이나 보조 장치로 탄환을 처리할 수 있다.
- 적 탄환은 공과 빌드 효과와 색뿐 아니라 형태와 궤적으로도 구분한다.
- 화면 밖 기습과 회피 불가능한 탄막을 만들지 않는다.

### 4.5 임시 분열 공

- 충전된 공이 적을 맞힐 때 임시 분열 공을 생성하는 능력이 존재한다.
- 벽 충돌은 기본 분열 발동 조건으로 사용하지 않는다.
- 임시 공은 영구 공 6개 제한을 넘을 수 있다.
- 임시 공은 낮은 피해를 주며 기본 약 2초 뒤 소멸한다.
- 기본 임시 공은 다시 분열하지 않는다.
- 최종 진화는 제한된 2차 분열을 한 번 허용할 수 있다.
- 임시 공은 동시에 최대 30개까지 존재하며 객체 재사용 풀로 관리한다.

후반 분열 빌드의 목표 장면은 적 무리 안에서 다수의 공이 짧게 연쇄 반사하며 화면을 정리하는 순간이다. 궤적, 연속 타격음, 처치음 상승으로 쾌감을 전달한다.

## 5. 능력과 빌드

### 5.1 슬롯

- 주 능력 5칸: 공 변형 또는 자동 보조무기
- 보조 모듈 3칸: 공격, 충전, 패들, 방어 강화
- 보스 유물 2칸: 일반 슬롯과 별도
- 같은 능력을 다시 선택하면 최대 5등급까지 강화
- 슬롯이 차면 새 능력 대신 기존 강화와 보조 모듈을 제시
- 모든 슬롯과 등급이 최대라면 공격력 증가, 보호막 회복, 점수 보너스 중 하나를 반복 선택

슬롯 수와 최대 등급은 플레이테스트 가설이다. 선택 피로와 빌드 완성 속도를 기준으로 조정한다.

### 5.2 주 능력 계열

- 폭발: 충돌 지점 범위 피해
- 관통: 반사 전에 여러 적 통과
- 분열: 임시 공 생성
- 전기: 주변 적 연쇄 피해
- 미사일: 공이 맞힌 적을 자동 추적
- 레이저: 패들에서 주기적으로 수직 발사
- 드론: 자동 공격 또는 적 탄환 요격

모든 보조무기는 자동으로 동작한다. 추가 발사 버튼은 만들지 않는다.

### 5.3 보조 모듈 계열

- 공 피해와 속도
- 충전 횟수와 충전 피해
- 패들 폭, 최대 체력, 보호막
- 폭발 범위와 상태 효과 지속시간
- 자동 무기 재사용 대기시간

### 5.4 진화

진화에는 다음 조건이 필요하다.

1. 주 능력 최대 등급
2. 대응 보조 모듈 보유
3. 호환 보스 유물 획득

보스 유물 선택지 3개 중 최소 1개는 현재 빌드와 연결되어야 한다. 무작위 선택 때문에 진화 가능성이 완전히 사라지지 않게 한다.

예시 진화:

- 분열 5등급
- 충전 모듈 보유
- 증식 유물 획득
- 결과: 충전 공 적중 시 임시 공이 대량 생성되고 한 번의 2차 분열 허용

진화는 수치만 오르는 것이 아니라 공, 패들 부품, 궤적, 충돌 효과가 확실히 변하는 사건으로 표현한다.

## 6. 적과 보스

### 6.1 첫 스테이지 일반 적

1. 기본형: 느리게 하강하며 공격하지 않는다.
2. 장갑형: 체력이 높고 느리다.
3. 사수형: 일정 주기로 패들 방향 탄환을 발사한다.
4. 확산형: 부채꼴 탄환을 발사한다.
5. 기동형: 좌우로 움직이며 하강한다.
6. 방어형: 전면 방어막이 있고 측면과 후면 반사 공격이 효과적이다.

### 6.2 엘리트

- 일반 적의 크기, 체력, 패턴을 강화한 형태다.
- 추가 탄환 패턴을 사용하거나 주변 적을 강화한다.
- 처치하면 특별 아이템을 확정 드롭한다.

### 6.3 중간 보스

- 큰 사각형 실루엣을 유지한다.
- 미사일, 낙하물, 조준 사격 중 2개 패턴을 조합한다.
- 일반 적보다 넓게 움직이되 공의 궤적을 읽을 공간을 남긴다.
- 처치하면 보스 유물 3개 중 1개를 선택한다.

### 6.4 최종 보스

- 여러 사각 부위로 구성된 대형 적이다.
- 외장 부위를 파괴해야 핵이 노출된다.
- 측면 약점은 공 반사를 활용하면 효율적으로 공격할 수 있다.
- 체력 단계에 따라 공격 패턴이 변한다.
- 보스전 중 일반 적은 소수만 등장한다.
- 처치하면 스테이지가 끝난다.

## 7. 스테이지와 장기 난이도

- 런 시작 전에 스테이지를 선택한다.
- 각 스테이지는 전용 적 조합, 환경 규칙, 보스, 배경을 가진다.
- 첫 스테이지는 특수 환경 없이 기본 규칙을 가르친다.
- 이후 스테이지는 이동 벽, 위험 구역, 탄환 변형 같은 규칙을 한 번에 하나씩 추가한다.
- 스테이지를 클리어하면 다음 스테이지와 상위 위협도를 해금한다.

위협도는 단순 체력 배율이 아니다. 단계별로 다음 요소를 추가한다.

- 적 하강 속도
- 엘리트 빈도
- 적 조합의 복잡도
- 새 탄환 패턴
- 보스 기술

각 패들, 스테이지, 위협도 조합의 최고 기록과 클리어 빌드를 저장한다.

## 8. 영구 진행과 반복 플레이

영구 진행은 기본 능력치를 올리지 않는다. 새 선택지와 표현을 연다.

### 8.1 해금 대상

- 새 패들
- 새 주 능력과 보조 모듈
- 새 보스 유물
- 새 스테이지
- 상위 위협도
- 패들과 공 외형, 색상, 궤적 효과

### 8.2 해금 조건

- 특정 스테이지 클리어
- 특정 패들로 보스 처치
- 특정 능력 진화
- 특정 빌드 도전 달성
- 상위 위협도 클리어

### 8.3 장기 반복 축

- 스테이지별 위협도 상승
- 패들별 시작 특성과 전용 능력
- 능력 조합과 진화 수집
- 고정 조건 도전 모드
- 패들별 숙련도와 최고 기록
- 클리어 빌드 기록

패들 숙련도는 외형과 연출만 해금한다. 모든 콘텐츠를 해금한 뒤에도 위협도, 기록, 도전은 남는다. 끝없는 생존, 고정 시드 도전, 온라인 순위표는 후속 확장이다.

## 9. 패들 차별화

모든 패들은 같은 좌우 드래그를 사용한다. 각 패들은 다음을 가진다.

- 고유 시작 특성
- 전용 능력 1개
- 빌드 성향을 보여주는 패들과 공 표현

예시 방향:

- 돌격형: 패들로 친 공의 충전 피해 강화
- 분열형: 임시 공 생성과 지속 강화
- 요격형: 적 탄환 방어와 보호막 강화

첫 플레이 버전은 기본 패들 1개만 구현한다.

## 10. 화면과 조작

### 10.1 화면 배치

- 상단: 진행 시간과 보스 체력
- 중앙: 적, 공, 탄환 전투 영역
- 하단 위쪽: 패들
- 최하단: 패들 체력, 경험치, 보스 유물
- 능력 목록은 평소 작게 표시하고 일시정지 화면에서 상세 표시

### 10.2 모바일

- 화면 하단 어디서든 좌우 드래그한다.
- 패들은 손가락 절대 위치가 아니라 상대 이동량을 따른다.
- 패들은 손가락보다 위에 있어 가려지지 않는다.
- 손을 떼어도 전투는 계속된다.
- 레벨업과 보스 유물 선택 때만 전투가 완전히 멈춘다.

### 10.3 데스크톱

- 마우스 좌우 이동을 지원한다.
- `A/D`와 방향키를 지원한다.
- 중앙에 세로 전투 영역을 배치한다.
- 화면 크기와 무관하게 논리 전투장 너비와 물리는 동일하다.

## 11. 시각·청각 피드백 원칙

정식 미술 스타일과 세계관은 코어 프로토타입 이후 정한다. 첫 버전은 사각형 적과 기하학적 임시 그래픽을 사용하되 전투 판독성과 성장 표현을 검증한다.

- 공 속성에 따라 색, 코어 모양, 궤적, 충돌 효과 변경
- 관통, 폭발, 충전, 분열 상태를 즉시 구분
- 자동 무기는 패들에 장착된 부품으로 표시
- 능력 등급이 오르면 효과가 점진적으로 커짐
- 진화 시 외형과 효과가 확실히 변함
- 패들 정타에 전용 타격음, 짧은 진동, 섬광 제공
- 연속 처치에 단계적으로 높아지는 효과음 제공
- 화면 흔들림, 번쩍임, 진동 강도 설정 제공

## 12. 첫 플레이 버전

### 12.1 포함

- Phaser, TypeScript, Vite 기반 웹 게임
- 세로 화면 모바일·데스크톱 실행
- 기본 패들 1개
- 스테이지 1개
- 일반 적 6종
- 중간 보스 2종
- 최종 보스 1종
- 주 능력 7종
- 보조 모듈 5종
- 보스 유물 6종
- 능력 등급과 최소 3개 진화
- 영구 공 최대 6개와 임시 분열 공
- 경험치, 레벨업, 3개 선택지
- 낙하 회복과 보호막 아이템
- 체력, 피격, 실패, 클리어
- 설정, 최고 기록, 클리어 여부 로컬 저장
- 상위 위협도 1단계 해금
- 기본 사운드, 진동, 전투 효과
- 데이터 기반 밸런스 설정

### 12.2 제외

- 추가 패들과 스테이지
- 대규모 해금 트리
- 끝없는 모드
- 일일·주간 도전
- 온라인 순위표와 계정
- 결제와 광고
- 가로 화면 전용 UI
- 정식 아트, 스토리, 세계관
- 복잡한 튜토리얼
- 모든 능력의 진화

## 13. 기술 설계

### 13.1 기술 선택

- Phaser: 2D 렌더링, 입력, 장면, Arcade Physics
- TypeScript: 게임 규칙과 데이터 타입
- Vite: 개발 서버와 웹 빌드
- Canvas: 전투 렌더링
- HTML UI: 메뉴와 설정
- 브라우저 로컬 저장: 설정, 기록, 해금

### 13.2 장면

- `Boot`: 자원 로딩과 저장 데이터 검증
- `Menu`: 시작, 설정, 기록
- `Run`: 전투
- `Upgrade`: 능력과 보스 유물 선택
- `Result`: 클리어, 실패, 빌드 기록

### 13.3 시스템 경계

- 입력: 터치, 마우스, 키보드를 공통 이동 명령으로 변환
- 공 물리: 반사각, 속도 제한, 충전, 영구·임시 공
- 적: 이동, 공격 패턴, 최하단 도달
- 전투: 피해, 폭발, 관통, 상태 효과
- 런 진행: 시간, 적 생성, 중간 보스, 최종 보스
- 빌드: 능력 등급, 슬롯, 진화, 유물
- 보상: 경험치, 레벨업, 낙하 아이템
- 저장: 설정, 최고 기록, 해금 상태
- 표현: 이펙트, 사운드, 진동

표현 시스템은 전투 판정을 변경하지 않는다. 능력, 적, 웨이브 수치는 코드에 흩어 놓지 않고 설정 데이터로 분리한다.

### 13.4 주요 데이터 흐름

`적 처치 -> 경험치 획득 -> 레벨업 -> 전투 정지 -> 선택 -> 빌드 수정 -> 전투 재개`

`적 최하단 도달 -> 적 제거 -> 패들 피해 -> 체력 확인 -> 계속 또는 런 실패`

`중간 보스 처치 -> 전투 정지 -> 보스 유물 선택 -> 진화 조건 확인 -> 전투 재개`

### 13.5 안정성

- 임시 공, 탄환, 이펙트는 객체 재사용 풀 사용
- 임시 공과 탄환에 최대 개수 제한
- 브라우저가 백그라운드로 가면 자동 일시정지
- 저장 데이터에 스키마 버전 부여
- 손상되거나 구버전인 저장은 보존 가능한 필드만 읽고 안전한 기본값 사용
- 고정 논리 해상도로 화면 크기 변경이 전투 좌표를 바꾸지 않게 함
- 자원 로딩 실패 시 핵심 전투에 필요한 단순 대체 그래픽 사용

## 14. 검증 계획

### 14.1 자동 검증

- 반사각과 최소 수직각
- 충전 횟수와 피해 계산
- 관통, 폭발, 분열의 발동 순서
- 임시 공 수명과 최대 개수
- 능력 슬롯, 등급, 진화 조건
- 피해, 무적 시간, 보호막 순서
- 12분 웨이브와 보스 출현 시점
- 저장, 불러오기, 손상 복구
- 터치, 마우스, 키보드 입력 변환

### 14.2 직접 검증

- 모바일 화면 크기별 UI와 손가락 가림 확인
- 공, 적 탄환, 낙하 아이템 구분 확인
- 영구 공 6개와 임시 공 대량 생성 시 성능 확인
- Chrome과 Safari 계열 실제 모바일 확인
- 5~10회의 전체 런으로 빌드 속도와 난이도 확인

### 14.3 성능 목표

- 일반 전투 60fps 목표
- 대량 분열 순간에도 패들 입력 지연 없이 유지
- 저사양 이펙트 감소가 전투 판정을 바꾸지 않음

### 14.4 재미 검증 기준

- 처음 보는 플레이어가 설명 없이 패들을 움직인다.
- 패들로 공을 받아치는 이유를 체감한다.
- 초반과 후반의 화력 차이가 명확하다.
- 분열 진화가 의도한 순간 화면 정리 경험을 만든다.
- 실패 원인을 이해할 수 있다.
- 한 판 뒤 다른 빌드로 다시 플레이할 의향이 생긴다.

이 기준을 만족하지 못하면 콘텐츠를 추가하기 전에 조작, 충전 보상, 성장 속도, 적 밀도를 먼저 수정한다.
