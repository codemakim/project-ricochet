# Enemy Descent Tuning Design

## 목적

첫 사용자 플레이테스트에서 코어 루프는 이전 프로토타입보다 재미있고 게임답다는 평가를 받았다. 남은 즉시 조정점은 적 대형이 내려오는 압박이 약간 빠르다는 점이다.

## 변경

- 모든 프로토타입 일반 적의 하강 속도를 `26px/s`에서 `22px/s`로 낮춘다.
- 약 15% 감속이다.
- 기본 대형 20마리, 적 체력, shooter 수, 1300ms 발사 주기, 350ms warning, 180px/s 적 탄환 속도는 바꾸지 않는다.
- 플레이어·구슬·회수·반사·실험 설정은 바꾸지 않는다.

현재 대형에서 가장 아래 행이 breach 지점에 도달하는 시간은 약 20.5초에서 약 24.3초로 늘어난다. 가장 위 행은 약 27초에서 약 32초로 늘어난다. 적 수와 공격 빈도를 유지하므로 화면 밀도와 슈팅 압박은 남고, 첫 breach 대응 시간만 늘어난다.

## 구현 경계

- `src/game/enemies/enemyRules.ts`의 고정 formation 속도만 `22`로 변경한다.
- formation 규칙 테스트와 `EnemyManager` 생성 속도 테스트의 기대값을 `22`로 변경한다.
- 별도 난이도 곡선, 설정 UI, 적 종류별 속도 분기는 추가하지 않는다.

## 검증

- TDD로 기존 `26px/s` 기대 테스트를 먼저 `22px/s`로 바꾸고 RED를 확인한다.
- 전체 unit, desktop/mobile E2E, production build를 다시 실행한다.
- 플레이테스트에서는 적이 여전히 움직이고 shooter/bullet 상한이 유지되는지 확인한다.
- 재미와 압박 평가는 사용자 재플레이 피드백으로 최종 판단한다.
