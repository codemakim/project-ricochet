# 연속 전투 진행 설계

## 1. 목적

현재 프로토타입은 고정된 적 20마리를 한 번 생성하고, 모두 제거되면 더 진행되지 않는다. 이를 화면 전환 없는 연속 전투로 확장한다.

플레이어에게는 웨이브 경계가 보이지 않는다. 적 대형은 기존 적이 남아 있어도 상단에서 계속 유입된다. 레벨업 선택과 보스 보상 선택 때만 전투가 완전히 정지한다.

이 문서는 연속 생성, 구간 난이도, 경험치 정지, 보스 진입과 다음 구간 재개의 흐름을 정의한다. 능력별 세부 수치와 보스 패턴은 기존 코어 리디자인 명세의 능력·보스 규칙을 따른다.

## 2. 플레이 흐름

1. 구간 시작 시 현재 프로토타입과 비슷한 20마리 대형을 생성한다.
2. 전투가 진행되는 동안 생성 시계가 다음 보충 대형을 준비한다.
3. 생성 간격, 상단 공간, 활성 적 상한을 모두 만족하면 6~10마리 보충 대형을 생성한다.
4. 적 처치 경험치는 별도 아이템 없이 즉시 누적한다.
5. 레벨업 조건을 만족하면 전투를 정지하고 능력 3개 중 1개를 선택한다.
6. 3분이 지나면 일반 적 신규 생성을 중단하고 보스 경고를 시작한다.
7. 남아 있는 일반 적을 삭제하지 않고 보스를 등장시킨다.
8. 보스 처치 시 전투를 정지하고 보스 유물 3개 중 1개를 선택한다.
9. 선택이 끝나면 위협도가 높아진 다음 3분 구간을 즉시 시작한다.

전멸은 다음 대형의 생성 조건이 아니다. 화면에는 웨이브 번호, 웨이브 클리어 문구, 구간 전환 화면을 표시하지 않는다.

## 3. 상태 모델

전투 진행 상태는 다음 다섯 가지다.

- `running`: 일반 대형을 연속 생성하고 구간 시계를 진행한다.
- `levelUpPaused`: 게임 시간을 정지하고 일반 능력을 선택한다.
- `bossWarning`: 일반 대형 생성을 중단하고 보스 진입 경고를 재생한다.
- `boss`: 일반 대형 생성을 중단하고 보스전을 진행한다.
- `bossRewardPaused`: 게임 시간을 정지하고 보스 유물을 선택한다.

패배는 별도 최상위 상태다. 플레이어 체력이 0이 되면 진행 상태와 관계없이 패배 처리가 우선한다.

허용 전이는 다음과 같다.

```text
running -> levelUpPaused -> running
running -> bossWarning -> boss
bossWarning -> levelUpPaused -> bossWarning
boss -> levelUpPaused -> boss
boss -> bossRewardPaused -> running
any state -> defeated
```

레벨업 정지는 직전 진행 상태를 `resumeState`로 보존한다. 선택 뒤 `running`, `bossWarning`, `boss` 중 정지 전 상태로 돌아간다. 레벨업과 3분 종료가 같은 프레임에 발생하면 레벨업 선택을 먼저 처리한다. 선택이 끝난 뒤 보스 경고와 보스 상태로 전환한다. 한 번에 여러 레벨을 얻으면 선택을 하나씩 연속 처리하며, 모든 선택이 끝날 때까지 전투를 재개하지 않는다.

## 4. 구성 요소

### 4.1 `EncounterDirector`

- 현재 구간 번호, 구간 경과 시간, 위협 단계, 진행 상태를 소유한다.
- 생성 가능 여부를 판단하고 `FormationSpawner`에 생성 요청을 보낸다.
- 3분 종료, 보스 경고, 보스 처치, 다음 구간 시작을 조정한다.
- 렌더링이나 개별 적 생명주기를 직접 처리하지 않는다.

### 4.2 `FormationSpawner`

- 구간 번호와 위협 단계를 받아 적 대형 명세를 만든다.
- 사각형, 벌집형, 지그재그 등 대형 템플릿에서 하나를 고른다.
- 기본 적, 장갑 적, 사수형 비율을 위협 단계에 맞춰 배치한다.
- 실제 적 생성은 `EnemyManager`에 위임한다.

### 4.3 `ProgressionManager`

- 적 종류별 경험치를 누적하고 레벨업 대기 횟수를 계산한다.
- 능력 선택지 3개를 만들고 선택 결과를 현재 빌드에 적용한다.
- 보스 처치 시 보스 유물 선택지 3개를 만든다.
- 보스 유물 선택지에는 현재 빌드와 연결되는 항목을 최소 1개 보장한다.

### 4.4 `GamePauseController`

- 레벨업과 보스 보상 중 물리, 구슬, 적, 탄환, 생성 시계, 구간 시계를 함께 정지한다.
- 선택 UI 입력과 렌더링은 계속 동작시킨다.
- 브라우저 탭 비활성화 같은 플랫폼 정지와 게임 선택 정지를 구분한다.

## 5. 연속 생성 규칙

보충 대형은 다음 조건을 모두 만족할 때만 생성한다.

1. 현재 상태가 `running`이다.
2. 생성 간격이 지났다.
3. 직전 대형의 최상단 적이 생성 안전 구역을 벗어나 새 대형과 겹치지 않는다.
4. 새 대형을 추가해도 현재 위협 단계의 활성 적 상한을 넘지 않는다.

조건이 충족되지 않으면 생성 요청을 버리지 않고 대기한다. 다음 업데이트에서 다시 검사하며, 조건이 충족되는 즉시 생성한다. 여러 대형을 한 프레임에 몰아서 생성하지 않는다.

보스가 직접 소환하는 약한 적은 일반 대형 생성 시계의 영향을 받지 않는다. 대신 보스 전용 소환 상한을 적용해 화면이 막히지 않게 한다.

## 6. 첫 구간 난이도

하강 속도보다 적의 수와 종류로 압박을 높인다. 첫 구간의 기본 하강 속도는 현재 플레이테스트에서 승인한 `22px/s`를 유지한다.

- 0~1분: 기본 적 중심, 활성 적 상한 22
- 1~2분: 장갑·사수 비율 증가, 활성 적 상한 26
- 2~3분: 더 큰 보충 대형과 짧은 생성 간격, 활성 적 상한 30

보스 처치 후 구간 번호가 오르면 적 체력, 특수 적 비율, 대형 복잡도를 먼저 높인다. 하강 속도는 구간 경계에서만 소폭 높일 수 있으며 `26px/s`를 넘지 않는다. 동시 사수와 적 탄환 상한은 활성 적 상한과 별도로 유지한다.

보충 대형 크기와 생성 간격은 한곳의 밸런스 설정으로 관리한다. 초기 보충 대형 크기는 6~10마리다. 첫 플레이테스트 값은 0~1분 8초, 1~2분 7초, 2~3분 6초다. 상단 공간과 활성 적 상한 조건은 이 시간이 지나도 그대로 적용한다. 이후 수치는 직접 플레이 결과로 조정하되, 동일 시드 자동 테스트에서는 같은 생성 순서를 재현해야 한다.

## 7. 경험치와 선택 정지

- 적이 죽는 순간 해당 경험치를 자동 획득한다.
- 경험치 오브젝트를 생성하거나 플레이어가 회수하지 않는다.
- 경험치 바가 가득 차면 해당 프레임의 피해와 처치 정산을 끝낸 뒤 `levelUpPaused`로 전환한다.
- 정지 중 조준 방향과 구슬 상태를 보존한다.
- 선택 완료 후 모든 객체가 정지 직전 속도와 상태로 재개된다.
- 일반 능력은 기존 코어 리디자인 명세의 주 능력, 재발사·충전 능력, 회수 능력 풀에서 제공한다.

경험치 요구량과 적 종류별 경험치는 밸런스 데이터로 둔다. 진행 규칙 테스트는 특정 수치 대신 `경험치 누적 -> 대기 레벨 계산 -> 선택 횟수 소비` 계약을 검증한다.

## 8. 보스 진입과 다음 구간

- 구간 시계가 180초에 도달하면 일반 대형 생성 요청을 중단한다.
- 이미 화면에 있는 일반 적과 적 탄환은 유지한다.
- 짧은 시각·음향 경고 동안 구간 시계는 더 진행하지 않는다.
- 경고 뒤 기존 코어 리디자인 명세의 중간 보스를 생성한다.
- 보스는 자체 규칙으로 약한 적 대형을 소환할 수 있다.
- 보스 처치 시 남은 보스 소환물과 적 탄환을 정리하고 `bossRewardPaused`로 전환한다.
- 유물 선택 후 구간 번호를 1 올리고 일반 생성 시계와 3분 구간 시계를 초기화한다.
- 플레이어, 체력, 현재 빌드, 영구 구슬 상태는 유지한다.

첫 구현은 중간 보스 반복 구조로 진행 루프를 검증한다. 최종 보스와 런 승리 조건은 스테이지·메타 진행 설계에서 별도로 다룬다.

## 9. 실패와 복구

- 활성 적 상한 때문에 생성이 지연되는 것은 정상 상태다.
- 생성 가능한 대형 템플릿이 없으면 가장 단순한 기본 적 대형으로 대체한다.
- 선택 UI가 열린 동안 추가 경험치나 보스 타이머가 진행되지 않는다.
- 패배가 발생하면 예약된 생성, 레벨업 대기, 보스 경고를 모두 중단한다.
- 재시작하면 구간 번호, 경험치, 빌드, 생성 시계를 초기 상태로 되돌린다.

## 10. 검증 기준

### 10.1 단위 테스트

- 생성 간격, 상단 공간, 활성 적 상한 중 하나라도 실패하면 생성하지 않는다.
- 대기한 생성은 조건 충족 후 한 번만 실행된다.
- 0~1분, 1~2분, 2~3분의 위협 단계와 활성 적 상한이 정확하다.
- 여러 레벨을 한 번에 얻으면 필요한 선택 횟수를 잃지 않는다.
- 레벨업과 3분 종료가 겹치면 레벨업을 먼저 처리한다.
- 보스 처치와 유물 선택 뒤 다음 구간 번호와 시계가 정확히 초기화된다.

### 10.2 브라우저 자동 테스트

- 일반 적을 전멸시키지 않아도 다음 대형이 상단에서 들어온다.
- 3분 동안 웨이브 전환 화면 없이 전투가 이어진다.
- 레벨업 선택 중 적, 탄환, 구슬, 구간 시계가 움직이지 않는다.
- 보스 경고 후 일반 대형이 더 생성되지 않는다.
- 보스 보상 선택 후 다음 구간의 일반 적 생성이 재개된다.

### 10.3 직접 플레이

- 화면이 비는 시간이 길지 않다.
- 활성 적이 많아도 조준 가능한 통로가 남는다.
- 난이도 상승이 하강 속도 급증보다 적 조합 변화로 느껴진다.
- 레벨업 정지가 전투 흐름을 정리하는 짧은 호흡으로 작동한다.
- 보스 진입이 별도 화면 전환 없이도 명확하게 인식된다.

## 11. 구현 순서

1. 연속 생성 규칙과 `EncounterDirector`
2. 여러 대형을 받을 수 있는 `EnemyManager`
3. 경험치 누적, 정지, 최소 능력 선택 수직 구현
4. 3분 구간과 보스 진입
5. 보스 보상과 다음 구간 재개
6. 전체 능력 풀, 보스 패턴, 정식 그래픽·사운드 확장

첫 개발 단위는 1~2번이다. 먼저 `고정 대형 전멸 후 정지`를 `연속 대형 유입`으로 바꾸고 플레이 감각을 확인한다. 경험치와 보스는 그 결과 위에 순서대로 붙인다.
